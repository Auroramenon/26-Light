## Python 端功能详解

### 1. 数据采集与处理流程

**文件位置**: `D:\guangdian\26-Light\python\`

#### 1.1 相机采集 (`capture/camera.py`)
- 使用工业相机 LRCP S400 (USB 3.0)
- 采集 30fps 的视频流
- 支持 RGB 和 NIR (近红外) 模式

#### 1.2 人脸检测与关键点提取
- **人脸检测** (`face/detector.py`): 使用 MediaPipe Face Mesh
- **关键点追踪** (`face/tracker.py`): 468 个面部关键点
- **ROI 提取** (`face/roi.py`): 提取额头区域用于 rPPG

#### 1.3 生理信号提取 (rPPG)
**文件**: `rppg/hr_estimator.py`, `rppg/hrv_analyzer.py`

- **BVP 信号提取**: 从额头 ROI 提取血容积脉搏信号
- **心率计算**: 通过频域分析估算 HR (40-200 BPM)
- **HRV 分析**: 计算心率变异性指标
  - RMSSD: 相邻心跳间期差值的均方根
  - SDNN: 心跳间期标准差
  - LF/HF: 低频/高频功率比

#### 1.4 行为特征检测
**文件**: `behavior/eye_detector.py`, `behavior/yawn_detector.py`, `behavior/head_pose.py`

- **PERCLOS (闭眼比例)**: 
  - 通过 EAR (Eye Aspect Ratio) 判断闭眼
  - 统计 60 秒内闭眼帧占比
  
- **哈欠检测**:
  - 通过 MAR (Mouth Aspect Ratio) 检测张嘴
  - 连续超过阈值 10 帧计为一次哈欠
  
- **头部姿态**:
  - 使用 solvePnP 解算头部俯仰角
  - 检测低头/点头动作

### 2. 疲劳判断算法

**文件**: `fusion/feature_fusion.py`, `fusion/fatigue_classifier.py`

#### 2.1 风险映射
将各项指标映射到 [0, 1] 的风险分：

```python
Risk_perclos = map(PERCLOS, 0.15, 0.30)  # 闭眼占比
Risk_yawn    = map(哈欠频率, 0.2, 1.0)    # 次/分钟
Risk_head    = map(|头部角度|, 15°, 35°)  # 低头角度
Risk_hrv     = normalize_hrv(RMSSD, SDNN, LF/HF)
```

#### 2.2 加权融合
```python
FatigueScore = 0.35 × Risk_hrv
             + 0.30 × Risk_perclos
             + 0.20 × Risk_yawn
             + 0.15 × Risk_head
```

#### 2.3 防抖机制
- **EMA 时间平滑**: 消除瞬时抖动
- **迟滞阈值**: 升级/降级使用不同阈值
- **持续性约束**: 必须持续 5 秒才切换等级

#### 2.4 疲劳等级输出
| 等级 | 名称 | 评分范围 | 含义 |
|------|------|----------|------|
| 0 | 正常 | < 0.30 | 状态良好 |
| 1 | 轻度疲劳 | 0.30-0.50 | 开始疲劳 |
| 2 | 中度疲劳 | 0.50-0.70 | 明显疲劳 |
| 3 | 重度疲劳 | ≥ 0.70 | 危险状态 |

### 3. 串口通信

**文件**: `comm/protocol.py`, `comm/serial_sender.py`

#### 3.1 数据包格式
```
$FL,<level>,<hr>,<checksum>\r\n
```

示例:
```
$FL,0,72,3F\r\n    # 正常, 心率72
$FL,1,68,35\r\n    # 轻度疲劳, 心率68
$FL,2,78,37\r\n    # 中度疲劳, 心率78
$FL,3,85,34\r\n    # 重度疲劳, 心率85
```

#### 3.2 校验算法
```python
body = f"FL,{level},{hr}"
checksum = 0
for c in body:
    checksum ^= ord(c)
packet = f"${body},{checksum:02X}\r\n"
```

## STM32 端功能详解

### 1. 硬件配置

**文件位置**: `D:\guangdian\26-Light\stm32\Light\`

#### 1.1 引脚分配
```
PA0  (TIM2_CH1) → WS2812B 数据线 (DIN)
PA6  (TIM3_CH1) → 蜂鸣器信号线
PA9  (USART1_TX) → USB-to-TTL RXD
PA10 (USART1_RX) → USB-to-TTL TXD
```

#### 1.2 外设配置
- **USART1**: 115200 波特率, 8N1
- **TIM2**: PWM 800kHz (WS2812B 驱动)
- **TIM3**: PWM (蜂鸣器控制)
- **DMA**: TIM2 PWM 数据传输

### 2. 软件模块

#### 2.1 串口协议解析 (`Core/Src/uart_protocol.c`)
- 中断接收 + 环形缓冲区
- 解析 `$FL,<level>,<hr>,<checksum>\r\n` 格式
- XOR 校验验证

#### 2.2 WS2812B 灯带驱动 (`Core/Src/ws2812b.c`)
- TIM2 PWM + DMA 驱动
- GRB 颜色编码 (每个 LED 24 bit)
- 支持 8 颗灯珠

**颜色映射**:
| 等级 | 颜色 | RGB 值 |
|------|------|--------|
| 0 | 绿色常亮 | (0, 255, 0) |
| 1 | 黄色常亮 | (255, 255, 0) |
| 2 | 橙色常亮 | (255, 128, 0) |
| 3 | 红色闪烁 | (255, 0, 0) 500ms 闪烁 |

#### 2.3 蜂鸣器控制 (`Core/Src/buzzer.c`)
- PWM 控制有源蜂鸣器
- 不同等级不同报警模式

**报警模式**:
| 等级 | 模式 |
|------|------|
| 0 | 静音 |
| 1 | 慢间歇 (1s 响 / 2s 停) |
| 2 | 快间歇 (300ms 响 / 300ms 停) |
| 3 | 持续报警 |

#### 2.4 疲劳反馈控制 (`Core/Src/fatigue_feedback.c`)
- 统一管理 LED 和蜂鸣器
- 根据疲劳等级调用相应的反馈模式

### 3. 主程序流程 (`Core/Src/main.c`)

```c
int main(void) {
    HAL_Init();
    SystemClock_Config();
    
    // 初始化外设
    MX_GPIO_Init();
    MX_DMA_Init();
    MX_TIM2_Init();
    MX_USART1_UART_Init();
    MX_TIM3_Init();
    
    // 初始化用户模块
    WS2812B_Init(&htim2, TIM_CHANNEL_1);
    Buzzer_Init(&htim3, TIM_CHANNEL_1);
    UART_Protocol_Init(&huart1);
    Feedback_Init();
    
    // 主循环
    while (1) {
        uint32_t now = HAL_GetTick();
        
        // 解析串口数据
        FatiguePacket pkt = UART_Protocol_Parse();
        if (pkt.valid) {
            Feedback_SetLevel((FatigueLevel)pkt.level);
        }
        
        // 更新反馈状态
        Feedback_Tick(now);
    }
}
```

## 数据流详解

### 完整数据流程

```
1. 相机采集 (30fps)
   ↓
2. MediaPipe 人脸检测 → 468 个关键点
   ↓
3. 并行处理:
   ├─→ 额头 ROI → rPPG → HR/HRV → Risk_hrv
   ├─→ 眼部关键点 → EAR → PERCLOS → Risk_perclos
   ├─→ 嘴部关键点 → MAR → 哈欠频率 → Risk_yawn
   └─→ 6 点 PnP → 头部姿态 → Risk_head
   ↓
4. 加权融合 → FatigueScore (0-1)
   ↓
5. EMA 平滑 + 迟滞判定 + 持续性约束
   ↓
6. 输出疲劳等级 (0/1/2/3) + 心率 (BPM)
   ↓
7. 串口发送: $FL,<level>,<hr>,<checksum>\r\n
   ↓
8. STM32 接收解析
   ↓
9. 并行输出:
   ├─→ WS2812B 灯带变色
   └─→ 蜂鸣器报警
```

### 时序说明

- **Python 端**: 每完成一轮 rPPG 分析发送一次 (约 1-2 秒/次)
- **STM32 端**: 实时接收，立即响应
- **LED 更新**: 收到新等级后立即更新颜色
- **蜂鸣器**: 根据等级持续运行相应模式

## 故障排查

### Python 端
- **相机打不开**: 检查 USB 3.0 连接, 尝试不同 `--camera` 索引
- **心率不准**: 确保光照均匀, 头部相对静止
- **串口连不上**: 检查 COM 口号, 确认波特率 115200

### STM32 端
- **灯带不亮**: 检查 DIN 方向, 确认 5V 供电
- **蜂鸣器不响**: 确认有源蜂鸣器, 检查 PA6 连接
- **串口无响应**: 检查 TX/RX 交叉连接, 确认 GND 共地

## 总结

本系统实现了从视觉采集到硬件反馈的完整闭环:

1. **Python 端**: 负责复杂的视觉处理和疲劳判断算法
2. **STM32 端**: 负责实时硬件控制和反馈
3. **串口通信**: 简单可靠的 ASCII 协议连接两端

通过多模态融合 (生理 + 行为) 和三级防抖机制，系统能够准确、稳定地检测疲劳状态并及时预警。
