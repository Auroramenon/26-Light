# 串口通信协议规范

## 概述

PC 端 Python 程序通过 USB-to-TTL 串口向 STM32 发送疲劳等级和心率数据。

## 物理层

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验位 | 无 |
| 停止位 | 1 |
| 接口 | USB-to-TTL (TX/RX/GND) |

## 数据包格式

### PC → STM32 (疲劳等级)

```
$FL,<level>,<hr>,<checksum>\r\n
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `$FL` | 固定头 | 数据包标识 |
| `level` | 0-3 | 疲劳等级: 0=正常, 1=轻度, 2=中度, 3=重度 |
| `hr` | 整数 | 心率 BPM (40-200) |
| `checksum` | 2位HEX | `FL,<level>,<hr>` 所有字符的 XOR 校验 |
| `\r\n` | 终止符 | 回车+换行 |

示例:
```
$FL,0,72,3F\r\n    -- 正常, 心率72
$FL,1,68,35\r\n    -- 轻度疲劳, 心率68
$FL,2,78,37\r\n    -- 中度疲劳, 心率78
$FL,3,85,34\r\n    -- 重度疲劳, 心率85
```

### 校验算法 (Python)

```python
body = f"FL,{level},{hr}"
checksum = 0
for c in body:
    checksum ^= ord(c)
packet = f"${body},{checksum:02X}\r\n"
```

### 校验算法 (C)

```c
uint8_t calc_checksum(const char *body) {
    uint8_t ck = 0;
    while (*body) ck ^= (uint8_t)*body++;
    return ck;
}
```

## 疲劳等级定义

| 等级 | 名称 | LED 颜色 | 蜂鸣器 | 触发条件 |
|------|------|----------|--------|----------|
| 0 | 正常 | 绿色常亮 | 静音 | 综合评分 < 0.3 |
| 1 | 轻度疲劳 | 黄色常亮 | 慢间歇 (1s/2s) | 0.3 ≤ 评分 < 0.5 |
| 2 | 中度疲劳 | 橙色常亮 | 快间歇 (300ms/300ms) | 0.5 ≤ 评分 < 0.7 |
| 3 | 重度疲劳 | 红色闪烁 | 持续报警 | 评分 ≥ 0.7 |

## 发送频率

PC 端每次完成一轮 rPPG 分析后发送一次，约每 1-2 秒一次。
