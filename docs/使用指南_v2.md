# 疲劳驾驶光电预警系统 — 使用指南 v2

## 一、系统总览

本系统由两部分组成：

- PC 端 Python 程序：通过工业相机采集面部视频，运行 rPPG 算法提取心率/HRV，结合行为特征（PERCLOS、哈欠、头姿）进行多模态融合，经 EMA 平滑 + 迟滞防抖 + 持续性约束后，输出四级疲劳等级。
- STM32 端固件：接收 PC 端的疲劳等级信号，驱动 WS2812B LED 灯带变色、蜂鸣器报警。

两者通过 USB-to-TTL 串口模块连接，协议为 ASCII 文本帧。

## 四、各元件杜邦线连接详解

### 4.1 USB-to-TTL 模块 ↔ STM32 (串口通信)

USB-to-TTL 模块一端插电脑 USB 口，另一端通过杜邦线连接 STM32。

```
USB-to-TTL 模块          STM32 Blue Pill
┌─────────────┐          ┌──────────────┐
│ TXD (发送)  │───白线──→│ PA10 (RX)    │
│ RXD (接收)  │───绿线──→│ PA9  (TX)    │
│ GND (地线)  │───黑线──→│ GND          │
│ VCC         │    不接   │              │
└─────────────┘          └──────────────┘
```

连接要点：

- TX 接 RX，RX 接 TX（交叉连接）
- GND 必须连接（共地），否则通信不稳定
- VCC 不要连接到 STM32 的 3.3V，STM32 通过自身 Micro-USB 供电
- 如果 USB-to-TTL 模块是 5V 电平的（如某些 CH340 模块），需确认 STM32 的 PA10 是否 5V 容忍（F103 的大部分引脚是 5V 容忍的，PA10 可以）

### 4.2 WS2812B LED 灯带 ↔ STM32

WS2812B 是单线协议，只需要一根数据线。

```
WS2812B 灯带              STM32 Blue Pill
┌──────────────┐          ┌──────────────┐
│ DIN (数据入) │───黄线──→│ PA0          │
│ VCC (+5V)    │───红线──→│ 5V           │
│ GND (地线)   │───黑线──→│ GND          │
└──────────────┘          └──────────────┘

灯带方向:  DIN ●━━●━━●━━●━━●━━●━━●━━● DOUT
           第1颗                    第8颗
```

连接要点：

- DIN 是灯带的数据输入端，注意灯带有方向（箭头指向数据流方向）
- 必须连接到灯带的 DIN 端，不是 DOUT 端
- VCC 接 5V 电源。如果灯珠较多（>8颗），建议使用独立 5V 电源，不要从 STM32 的 5V 引脚取电（电流不够）
- 建议在 DIN 线上串联一个 330Ω 电阻（保护第一颗灯珠）
- 建议在 VCC 和 GND 之间并联一个 100μF 电解电容（滤波稳压）

```
推荐电路:

PA0 ──[330Ω]──→ DIN
                  │
5V  ──────────→ VCC ──┤├── GND    (100μF 电解电容)
                  │         │
GND ──────────→ GND ───────┘
```

### 4.3 有源蜂鸣器模块 ↔ STM32

有源蜂鸣器模块通常有 3 个引脚：VCC、GND、I/O（信号）。

```
蜂鸣器模块                STM32 Blue Pill
┌──────────────┐          ┌──────────────┐
│ I/O (信号)   │───蓝线──→│ PA6          │
│ VCC (+5V)    │───红线──→│ 5V           │
│ GND (地线)   │───黑线──→│ GND          │
└──────────────┘          └──────────────┘
```

连接要点：

- I/O 引脚接高电平时蜂鸣器发声，低电平时静音
- 程序通过 PWM 控制 I/O 引脚的高低电平切换，实现间歇报警效果
- 如果蜂鸣器模块自带三极管驱动电路（大多数模块都有），可以直接连接
- 如果是裸蜂鸣器（无驱动板），需要外接一个 NPN 三极管（如 S8050）驱动

### 4.4 工业相机 ↔ 电脑

```
LRCP S400 工业相机
┌──────────────┐          ┌──────────────┐
│ USB 3.0 接口 │══USB线══→│ 电脑 USB 3.0 │
└──────────────┘          └──────────────┘
```

- 直接通过 USB 3.0 数据线连接电脑
- 确保插在电脑的 USB 3.0 接口上（蓝色接口），以保证带宽
- 首次使用可能需要安装相机驱动

## 五、供电方案

### 方案 A：USB 统一供电（推荐调试阶段）

- STM32 通过 Micro-USB 连接电脑供电（5V/500mA）
- WS2812B 和蜂鸣器从 STM32 的 5V 引脚取电
- 适用于 LED 数量 ≤ 8 颗的情况

### 方案 B：独立电源供电（推荐正式部署）

- STM32 通过 Micro-USB 供电
- WS2812B 灯带使用独立 5V/2A 电源供电
- 所有模块的 GND 必须连在一起（共地）

```
独立供电接线:

5V/2A 电源 ──→ WS2812B VCC
             ──→ 蜂鸣器 VCC
             ──→ GND (与 STM32 GND 连接)
```

## 六、串口使用说明

### 6.1 确认串口号

Windows 系统：

1. 将 USB-to-TTL 模块插入电脑
2. 打开「设备管理器」→「端口 (COM 和 LPT)」
3. 找到类似 `USB-SERIAL CH340 (COM3)` 的条目
4. 记下 COM 号（如 COM3）

如果看不到设备，需要安装对应芯片的驱动：

- CH340：搜索「CH340 驱动」下载安装
- CP2102：搜索「CP2102 驱动」下载安装
- FT232：搜索「FTDI 驱动」下载安装

### 6.2 串口参数

| 参数   | 值        |
| ------ | --------- |
| 波特率 | 115200    |
| 数据位 | 8         |
| 校验位 | 无 (None) |
| 停止位 | 1         |
| 流控制 | 无        |

### 6.3 串口调试

可以使用串口调试助手（如 SSCOM、PuTTY、或 VSCode 的 Serial Monitor）手动测试 STM32：

```
发送以下文本测试各等级响应:

$FL,0,72,3F\r\n    -- 正常, 心率72
$FL,1,68,35\r\n    -- 轻度疲劳, 心率68
$FL,2,78,37\r\n    -- 中度疲劳, 心率78
$FL,3,85,34\r\n    -- 重度疲劳, 心率85
```

注意：发送时需要勾选「发送新行」或手动添加 `\r\n` 结尾。

### 6.4 协议格式

```
$FL,<等级>,<心率>,<校验>\r\n

字段说明:
  $FL      — 固定包头
  等级     — 0/1/2/3
  心率     — 整数 BPM (40-200)
  校验     — "FL,<等级>,<心率>" 所有字符 XOR 后的 2 位十六进制
  \r\n     — 回车换行结尾
```

Python 端校验计算示例：

```python
body = f"FL,{level},{hr}"
checksum = 0
for c in body:
    checksum ^= ord(c)
packet = f"${body},{checksum:02X}\r\n"
```

## 八、疲劳判据详解（核心算法）

本节是系统的核心。系统采集四路信号，各自计算风险分 Risk ∈ [0, 1]，加权融合后经三级防抖输出最终疲劳等级。

### 8.1 数据流总览

```
相机帧 ──→ MediaPipe Face Mesh ──┬──→ EAR  ──→ PERCLOS ──→ Risk_perclos ──┐
                                 ├──→ MAR  ──→ 哈欠频率 ──→ Risk_yawn    ──┤
                                 ├──→ solvePnP ──→ 头部俯仰 ──→ Risk_head──┤
                                 └──→ ROI  ──→ rPPG ──→ HRV ──→ Risk_hrv ──┤
                                                                            │
                                    加权融合 (Σ w_i × Risk_i) ─────────────┘
                                         │
                                    EMA 时间平滑
                                         │
                                    迟滞阈值判定
                                         │
                                    持续性约束 (≥ K 秒)
                                         │
                                    输出疲劳等级 0/1/2/3
```

### 8.2 四路指标的风险映射

每个原始指标都通过分段线性函数映射到 [0, 1] 的风险分。

#### (1) PERCLOS — 闭眼时间占比

PERCLOS = 过去 60 秒内闭眼帧数 / 总帧数。

闭眼判定：EAR (Eye Aspect Ratio) < 阈值。

```
EAR 计算:
  左眼: landmarks [362, 385, 387, 263, 373, 380]
  右眼: landmarks [33, 160, 158, 133, 153, 144]
  EAR = (上下眼睑距离1 + 上下眼睑距离2) / (2 × 左右眼角距离)
  EAR < 0.2 → 判定为闭眼
```

风险映射：

```
Risk_perclos:
  PERCLOS < 0.15  →  0        (正常眨眼)
  0.15 ≤ p < 0.30 →  (p - 0.15) / 0.15   (线性过渡)
  PERCLOS ≥ 0.30  →  1        (严重闭眼)
```

直觉理解：闭眼占比不到 15% 是正常眨眼；超过 30% 就是严重嗜睡；中间线性过渡。

#### (2) 哈欠频率 — 每分钟哈欠次数

通过 MAR (Mouth Aspect Ratio) 检测张嘴动作。

```
MAR 计算:
  上唇: landmark 13, 下唇: landmark 14
  嘴角: landmarks 78 (左), 308 (右)
  MAR = 上下唇距离 / 左右嘴角距离
  MAR > 0.6 且持续 ≥ 10 帧 → 计为一次哈欠
```

风险映射：

```
Risk_yawn:
  rate < 0.2 次/分  →  0      (偶尔张嘴不算)
  0.2 ≤ rate < 1.0  →  (rate - 0.2) / 0.8  (线性过渡)
  rate ≥ 1.0 次/分  →  1      (频繁哈欠)
```

直觉理解：每分钟不到 0.2 次哈欠是正常的；超过 1 次/分钟就是严重疲劳信号。

#### (3) 头部俯仰 — 点头/低头检测

通过 solvePnP 解算头部 pitch 角度。

```
6 个关键点: 鼻尖、下巴、左右眼角、左右嘴角
pitch > 0 → 低头, pitch < 0 → 抬头
|pitch| > 15° → 异常头姿
```

风险映射：

```
Risk_head:
  |pitch| ≤ 15°  →  0
  15° < |pitch| < 35°  →  (|pitch| - 15) / 20  (线性过渡)
  |pitch| ≥ 35°  →  1
```

直觉理解：头部偏转 15° 以内是正常驾驶姿态；超过 35° 基本就是打瞌睡低头了。

#### (4) HRV — 心率变异性

通过 rPPG 提取的 BVP 信号计算。HRV 降低 = 自主神经调节能力下降 = 疲劳。

```
HRV 特征提取:
  1. BVP 信号带通滤波 (0.7-4.0 Hz)
  2. 峰值检测 → IBI (心跳间期)
  3. 计算:
     - RMSSD: 相邻 IBI 差值的均方根 (正常 20-50ms, 疲劳 <20ms)
     - SDNN:  IBI 标准差 (正常 50-100ms, 疲劳 <30ms)
     - LF/HF: 低频/高频功率比 (正常 0.5-2.0, 疲劳 >3.0)
```

HRV 风险分（三个子指标加权）：

```
rmssd_score = clip(1.0 - (rmssd - 15) / 25, 0, 1)
sdnn_score  = clip(1.0 - (sdnn  - 25) / 55, 0, 1)
lfhf_score  = clip((lf_hf - 1.0) / 3.0, 0, 1)

Risk_hrv = 0.4 × rmssd_score + 0.3 × sdnn_score + 0.3 × lfhf_score
```

直觉理解：RMSSD 越低、LF/HF 越高，说明副交感神经活性下降，疲劳风险越大。

### 8.3 加权融合

```
FatigueScore_raw = 0.35 × Risk_hrv
                 + 0.30 × Risk_perclos
                 + 0.20 × Risk_yawn
                 + 0.15 × Risk_head
```

权重设计依据：
- HRV (0.35)：生理金标准，不受主观控制，权重最高
- PERCLOS (0.30)：国际公认的驾驶疲劳核心指标
- 哈欠 (0.20)：直观但可被抑制，权重适中
- 头姿 (0.15)：辅助指标，容易受驾驶动作干扰，权重最低

### 8.4 EMA 时间平滑（消除瞬时抖动）

原始融合分逐帧计算，波动较大。用指数移动平均 (EMA) 平滑：

```
FatigueScore = α × FatigueScore_raw + (1 - α) × FatigueScore_prev

α = 0.3 (推荐值)
```

α 越小越平滑（反应越慢），α 越大越灵敏（抖动越多）。0.3 是平衡点。

实际效果：一次偶然闭眼不会让评分突然飙升，需要持续异常才会拉高分数。

### 8.5 迟滞阈值（防止等级来回跳）

如果升级和降级用同一个阈值，评分在阈值附近波动时等级会频繁切换（0→1→0→1...）。

解决方案：升级用较高阈值，降级用较低阈值。

```
升级阈值 (从低等级 → 高等级):  [0.30, 0.50, 0.70]
降级阈值 (从高等级 → 低等级):  [0.25, 0.45, 0.65]
```

示例：
- 评分从 0.28 升到 0.31 → 升级到 Level 1（超过升级阈值 0.30）
- 评分从 0.31 降到 0.29 → 仍然保持 Level 1（没有低于降级阈值 0.25）
- 评分降到 0.24 → 才降回 Level 0

### 8.6 持续性约束（防止瞬时误判）

即使评分超过了阈值，也必须持续 K 秒才真正切换等级。

```
持续性要求: K = 5 秒 (推荐值)
```

示例：评分在第 0 秒超过 0.70，但第 3 秒又降回 0.65 → 不会升级到 Level 3（没有持续满 5 秒）。

### 8.7 Level 3 多证据要求（可选，更严格）

为避免单一指标异常导致误报重度疲劳，可启用多证据模式：

```
Level 3 额外条件 (任选其一即可):
  - PERCLOS ≥ 0.30 (Risk_perclos = 1)
  - Risk_hrv ≥ 0.7
  - 同时满足: Risk_yawn ≥ 0.5 且 Risk_head ≥ 0.5
```

即：综合评分 ≥ 0.7 是必要条件，但还需要至少一路指标有强烈的疲劳证据。

| ---- | -------- | --------------- | --------------- | ---------------- | ----------------- |

| 1    | 轻度疲劳 | ≥ 0.30         | < 0.25 降回 0   | 黄色常亮         | 1秒响 / 2秒停     |
| 2    | 中度疲劳 | ≥ 0.50         | < 0.45 降回 1   | 橙色常亮         | 300ms响 / 300ms停 |
| 3    | 重度疲劳 | ≥ 0.70         | < 0.65 降回 2   | 红色闪烁 (500ms) | 持续报警          |

所有等级切换需持续 ≥ 5 秒才生效。

## 十一、常见问题排查

### 串口连不上

1. 检查设备管理器中是否识别到 COM 口
2. 确认 TX/RX 是否交叉连接（不是同名对接）
3. 确认 GND 是否连接
4. 确认波特率设置为 115200
5. 尝试更换 USB-to-TTL 模块或 USB 线

### WS2812B 灯带不亮

1. 检查 DIN 方向是否正确（灯带有箭头标识）
2. 检查 5V 供电是否正常（用万用表测量）
3. 检查 GND 是否与 STM32 共地
4. 尝试减少灯珠数量测试（先测 1 颗）
5. 检查 PA0 引脚是否正确连接

### 蜂鸣器不响

1. 确认是有源蜂鸣器（自带振荡电路，通电即响）
2. 用杜邦线直接将蜂鸣器 I/O 接 5V 测试是否能响
3. 检查 PA6 引脚连接
4. 确认蜂鸣器模块的 VCC 接了 5V

### 相机打不开

1. 确认相机 USB 线插在 USB 3.0 接口上
2. 尝试修改 `--camera` 参数（0, 1, 2 逐个试）
3. 检查是否安装了相机驱动
4. 用其他软件（如 AMCap）测试相机是否正常

### rPPG 心率不准

1. 确保面部光照均匀，避免强逆光
2. 保持头部相对静止（运动伪影会干扰信号）
3. 等待至少 8 秒让信号缓冲区填满
4. 尝试切换算法：`--method POS`
5. 如果使用近红外光源，确保 850nm LED 正常工作

### 疲劳等级频繁跳动

1. 确认 `ema_alpha` 设置为 0.3 或更低
2. 确认 `hold_seconds` ≥ 5
3. 确认迟滞阈值已启用（`upgrade_thresholds` 和 `downgrade_thresholds` 不同）
4. 如果仍然跳动，适当加大升级/降级阈值差距（如 0.08）

### HRV 数据不稳定

1. rPPG 提取的 HRV 精度低于接触式传感器，这是正常的
2. 确保面部 ROI 区域稳定（额头区域最佳）
3. 如果 HRV 持续不可靠，降低 `w_hrv` 权重，提高行为指标权重
