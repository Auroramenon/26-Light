# 串口连接问题诊断与解决方案

## 问题描述

Python 端疲劳检测功能正常，可以判断 level，但串口没有连接，导致 STM32 端的 LED 灯带没有颜色变化。

## 诊断结果

系统中**未检测到任何串口设备**，这意味着：
- USB-to-TTL 模块未连接，或
- STM32 未通过 USB 连接，或
- 驱动程序未安装

## 解决方案

### 方案 1: 连接 USB-to-TTL 模块 (推荐用于正式部署)

#### 1.1 硬件连接

```
USB-to-TTL 模块          STM32 Blue Pill
┌─────────────┐          ┌──────────────┐
│ TXD (发送)  │───────→│ PA10 (RX)    │
│ RXD (接收)  │───────→│ PA9  (TX)    │
│ GND (地线)  │───────→│ GND          │
│ VCC         │    不接   │              │
└─────────────┘          └──────────────┘

注意: TX 接 RX，RX 接 TX (交叉连接)
```

#### 1.2 检查设备管理器

1. 将 USB-to-TTL 模块插入电脑 USB 口
2. 打开「设备管理器」(Win+X → 设备管理器)
3. 展开「端口 (COM 和 LPT)」
4. 查找类似以下的设备：
   - `USB-SERIAL CH340 (COM3)`
   - `Silicon Labs CP210x USB to UART Bridge (COM4)`
   - `USB Serial Port (COM5)`

#### 1.3 安装驱动 (如果设备管理器中看不到)

根据你的 USB-to-TTL 芯片型号下载驱动：

**CH340/CH341 芯片**:
- 下载地址: http://www.wch.cn/downloads/CH341SER_EXE.html
- 或搜索: "CH340 驱动下载"

**CP2102/CP2104 芯片**:
- 下载地址: https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
- 或搜索: "CP2102 驱动下载"

**FT232 芯片**:
- 下载地址: https://ftdichip.com/drivers/vcp-drivers/
- 或搜索: "FTDI 驱动下载"

#### 1.4 配置串口号

确认串口号后，修改 `python/config.py`:

```python
# ===== 串口 =====
"serial_port": "COM3",        # 改为你实际的串口号
"serial_baud": 115200,
"serial_enabled": True,
```

### 方案 2: 直接通过 STM32 的 USB 口 (需要 USB CDC 固件)

如果你的 STM32 固件支持 USB CDC (虚拟串口)，可以直接通过 STM32 的 Micro-USB 口通信。

**注意**: 当前固件使用的是 USART1 (PA9/PA10)，不是 USB CDC。如果要使用这个方案，需要修改固件。

---

### 方案 3: 临时测试 - 使用虚拟串口对 (用于开发调试)

如果暂时没有硬件，可以使用虚拟串口进行测试。

#### 3.1 安装虚拟串口工具

**Windows**:
- 下载 com0com: https://sourceforge.net/projects/com0com/
- 或使用 Virtual Serial Port Driver (VSPD)

**配置**:
- 创建一对虚拟串口，例如 COM10 ↔ COM11
- Python 端连接 COM10
- 使用串口调试助手连接 COM11 查看数据

#### 3.2 修改配置

```python
"serial_port": "COM10",       # 虚拟串口
"serial_enabled": True,
```

## 常见错误及解决

### 错误 1: `[WinError 2] 系统找不到指定的文件`

**原因**: 串口号不存在或设备未连接

**解决**:
1. 运行 `python check_serial.py` 查看可用串口
2. 检查设备管理器
3. 确认硬件已连接

### 错误 2: `[WinError 5] 拒绝访问`

**原因**: 串口被其他程序占用

**解决**:
1. 关闭其他串口调试工具
2. 关闭 Arduino IDE / STM32CubeIDE
3. 重新插拔 USB 设备

### 错误 3: `[串口] 发送失败: [Errno 22] Invalid argument`

**原因**: 串口参数配置错误

**解决**:
1. 检查波特率是否为 115200
2. 检查数据位、停止位、校验位配置

### 错误 4: 串口连接成功但 LED 不变色

**原因**: 
- STM32 固件未烧录
- 串口协议不匹配
- 硬件连接错误

**解决**:
1. 确认 STM32 固件已烧录
2. 使用串口调试助手手动发送测试数据
3. 检查 TX/RX 是否交叉连接
4. 检查 STM32 的 USART1 配置 (PA9/PA10)

## 调试技巧

### 1. 启用详细日志

修改 `serial_sender.py`，在发送成功时也打印日志：

```python
# 在 send() 方法的 try 块中添加
self._port.write(packet)
print(f"[串口] 发送成功: Level={level}, HR={hr}, Packet={packet.hex()}")  # 添加这行
self._last_send_ts = now
```

### 2. 查看串口状态

在 `main.py` 主循环中添加：

```python
# 每 5 秒打印一次串口状态
if frame_count % (fps * 5) == 0:
    status = serial_out.status()
    print(f"[串口状态] {status}")
```

### 3. 使用串口监听工具

同时打开两个串口调试助手：
- 一个连接 Python 端的串口 (监听模式)
- 一个连接 STM32 端的串口 (监听模式)

这样可以看到双向的数据流。
