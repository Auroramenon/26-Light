# 串口问题解决 - 快速指南

## 问题现状

✅ Python 端疲劳检测正常，可以判断 level
❌ 串口未连接，LED 灯带没有颜色变化

## 诊断结果

系统中**未检测到串口设备**，需要连接硬件。

---

## 🚀 快速解决步骤

### 步骤 1: 运行诊断工具

**Windows 用户** (推荐):

```bash
# 双击运行
D:\guangdian\26-Light\python\diagnose_serial.bat
```

**或手动运行**:

```bash
cd D:\guangdian\26-Light\python
python check_serial.py
```

### 步骤 2: 根据诊断结果操作

#### 情况 A: 找到了串口设备

**示例输出**:

```
找到 1 个串口设备:

1. 端口: COM3
   描述: USB-SERIAL CH340 (COM3)
   硬件ID: USB VID:PID=1A86:7523

推荐使用: COM3
```

**操作**:

1. 记录串口号 (如 COM3)
2. 修改 `python/config.py`:
   ```python
   "serial_port": "COM3",    # 改为你的串口号
   "serial_enabled": True,
   ```
3. 运行测试:
   ```bash
   python test_serial.py
   ```
4. 如果测试通过，运行主程序:
   ```bash
   python main.py --serial COM3
   ```

#### 情况 B: 未找到串口设备

**输出**:

```
[警告] 未找到任何串口设备！
```

**原因**: 硬件未连接或驱动未安装

**解决方法** → 跳转到步骤 3

---

### 步骤 3: 连接硬件

#### 方案 A: 使用 USB-to-TTL 模块 (推荐)

**硬件连接**:

```
USB-to-TTL          STM32
  TXD    ────────→  PA10 (RX)
  RXD    ────────→  PA9  (TX)
  GND    ────────→  GND
  VCC    不连接
```

**注意**: TX 接 RX，RX 接 TX (交叉连接)

**检查连接**:

1. 将 USB-to-TTL 插入电脑 USB 口
2. STM32 通过 Micro-USB 供电
3. 确认 TX/RX/GND 三根线已连接

#### 方案 B: 使用 STM32 的 USB 口

**注意**: 当前固件使用 USART1，不支持 USB CDC。如需使用此方案需修改固件。

---

### 步骤 4: 安装驱动 (如果需要)

打开「设备管理器」(Win+X → 设备管理器)，查看「端口 (COM 和 LPT)」:

#### 看到串口设备 ✓

```
端口 (COM 和 LPT)
  └─ USB-SERIAL CH340 (COM3)
```

→ 驱动已安装，跳到步骤 5

#### 看到未知设备或黄色感叹号 ✗

```
其他设备
  └─ USB2.0-Serial (黄色感叹号)
```

→ 需要安装驱动

**根据芯片型号下载驱动**:

| 芯片型号      | 下载地址                                                              | 识别方法             |
| ------------- | --------------------------------------------------------------------- | -------------------- |
| CH340/CH341   | [官网](http://www.wch.cn/downloads/CH341SER_EXE.html)                    | 模块上印有 "CH340G"  |
| CP2102/CP2104 | [官网](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers) | 模块上印有 "CP2102"  |
| FT232         | [官网](https://ftdichip.com/drivers/vcp-drivers/)                        | 模块上印有 "FT232RL" |

**安装步骤**:

1. 下载对应驱动
2. 解压并运行安装程序
3. 重新插拔 USB 设备
4. 重新运行 `python check_serial.py`

### 步骤 5: 测试串口通信

```bash
cd D:\guangdian\26-Light\python
python test_serial.py
```

**测试内容**:

- 打开串口
- 发送 4 个测试数据包 (Level 0/1/2/3)
- 观察 STM32 响应

**预期结果**:

- ✓ 串口打开成功
- ✓ 数据包发送成功
- ✓ LED 灯带变色: 绿 → 黄 → 橙 → 红闪
- ✓ 蜂鸣器响起: 静音 → 慢间歇 → 快间歇 → 持续

**如果没有反应**:

1. 检查 STM32 固件是否已烧录
2. 检查 TX/RX 是否交叉连接
3. 检查 GND 是否连接
4. 使用串口调试助手手动测试

## 📋 故障排查

### 问题 1: 串口打开失败

**错误**: `[WinError 2] 系统找不到指定的文件`

**原因**: 串口号不存在

**解决**:

1. 运行 `python check_serial.py` 确认串口号
2. 修改 `config.py` 中的 `serial_port`

---

### 问题 2: 串口被占用

**错误**: `[WinError 5] 拒绝访问`

**原因**: 串口被其他程序占用

**解决**:

1. 关闭串口调试助手
2. 关闭 Arduino IDE / STM32CubeIDE
3. 重新插拔 USB 设备

---

### 问题 3: 串口连接成功但 LED 不变色

**可能原因**:

1. STM32 固件未烧录
2. TX/RX 未交叉连接
3. LED 灯带未连接或连接错误

**解决**:

1. 确认固件已烧录 (参考 `docs/编译问题解决记录.md`)
2. 检查接线:
   - USB-to-TTL TXD → STM32 PA10
   - USB-to-TTL RXD → STM32 PA9
   - USB-to-TTL GND → STM32 GND
3. 检查 LED 灯带:
   - DIN → STM32 PA0
   - VCC → 5V
   - GND → GND
