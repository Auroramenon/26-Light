# 香橙派自启动与现场演示方案

> 目标：插电即用，零键鼠、零电脑、零SSH、零密码输入
> 适用：现场演示、嵌入式部署、量产原型

---

## 一、整体目标与核心原理

### 1.1 期望的演示体验

```
出门前：
  ☑ 在家把外壳充满电
  ☑ 把外壳放包里

到现场：
  1. 拿出外壳
  2. 拨一下侧面开关
  3. 等 30 秒
  4. 小屏亮起，开始演示

✅ 零开盖  ✅ 零线缆  ✅ 零键鼠  ✅ 零手机操作
```

### 1.2 核心原理

```
香橙派开机流程：
内核启动 → systemd 启动所有服务 → 桌面登录框出现（可无视）
              ↑                        
       程序在这里就已经跑起来了
```

**关键点**：systemd 服务由 root 在系统启动时直接拉起，**不经过任何用户会话**，所以：
- ❌ 不需要登录
- ❌ 不需要输密码
- ❌ 不需要进入桌面
- ✅ 插电即跑

桌面密码框是给"人"用的，**与"后台服务"无关**。

## 二、systemd 自启动配置（核心）

### 2.1 创建服务文件

在香橙派上执行（首次部署时通过 SSH 或键鼠登录）：

```bash
sudo nano /etc/systemd/system/fatigue.service
```

写入以下内容：

```ini
[Unit]
Description=Fatigue Detection System
After=network.target

[Service]
Type=simple
User=orangepi
WorkingDirectory=/home/orangepi/26-Light/python
ExecStart=/home/orangepi/26-Light/python/venv/bin/python main.py --no-gui --serial /dev/ttyUSB0
Restart=on-failure
RestartSec=5
StandardOutput=append:/home/orangepi/fatigue.log
StandardError=append:/home/orangepi/fatigue.log

[Install]
WantedBy=multi-user.target
```

### 2.2 字段说明

| 字段 | 作用 | 部署时需修改的地方 |
|------|------|------------------|
| `User` | 以哪个用户运行 | 改为你的实际用户名 |
| `WorkingDirectory` | 程序工作目录 | 改为你的实际项目路径 |
| `ExecStart` | 启动命令 | 改为你的实际 Python 路径 |
| `--serial /dev/ttyUSB0` | STM32 串口设备 | 视实际串口设备而定 |
| `Restart=on-failure` | 崩溃自动重启 | 保持不变 |
| `RestartSec=5` | 重启等待秒数 | 保持不变 |
| `StandardOutput` | 日志输出文件 | 保持不变 |

### 2.3 启用服务

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable fatigue.service

# 手动启动测试（不重启情况下）
sudo systemctl start fatigue.service

# 查看运行状态
sudo systemctl status fatigue.service

# 查看实时日志
journalctl -u fatigue.service -f
```

### 2.4 常用管理命令

| 命令 | 用途 |
|------|------|
| `sudo systemctl start fatigue` | 启动 |
| `sudo systemctl stop fatigue` | 停止 |
| `sudo systemctl restart fatigue` | 重启 |
| `sudo systemctl status fatigue` | 查看状态 |
| `sudo systemctl disable fatigue` | 关闭开机自启 |
| `journalctl -u fatigue -f` | 实时日志 |
| `tail -f ~/fatigue.log` | 查看程序日志 |

## 三、桌面系统处理

桌面登录框对自启动**没有影响**，但有三种处理方式：

### 3.1 三种方案对比

| 场景 | 操作 | 资源占用 | 启动时间 | 推荐场景 |
|------|------|---------|---------|---------|
| **A. 保留桌面，不登录** | 啥都不改 | 桌面占 500MB 内存 | 30 秒 | 演示阶段，最省事 |
| **B. 保留桌面，自动登录** | 配置 lightdm | 桌面占 500MB 内存 | 30 秒 | 偶尔插显示器调试 |
| **C. 禁用桌面，CLI 模式** | 切换 multi-user.target | 仅占 200MB | 15 秒 | 量产部署，最推荐 |

### 3.2 方案 A：什么都不改

systemd 配好后，登录框停在那也没人管，程序在后台正常运行。**默认就是这样**。

### 3.3 方案 B：配置桌面自动登录

```bash
sudo nano /etc/lightdm/lightdm.conf
```

找到并修改：

```ini
[Seat:*]
autologin-user=orangepi
autologin-user-timeout=0
```

重启后桌面也会自动进入，不需要输密码。

### 3.4 方案 C：禁用桌面（推荐生产部署）

```bash
# 切换到 CLI 模式（不删除桌面，只是不启动）
sudo systemctl set-default multi-user.target

# 重启
sudo reboot
```

效果：
- ✅ 没有桌面界面，启动更快
- ✅ 节省 500MB 内存
- ✅ 服务正常运行，小屏正常显示

**恢复桌面**：

```bash
sudo systemctl set-default graphical.target
sudo reboot
```

## 四、硬件电源设计（外露开关）

### 4.1 设计目标

充电宝/电池**硬装在 3D 打印外壳内**，外壳侧面留一个开关孔，现场只需拨开关即可启动，**不需要开盖**。

### 4.2 推荐方案：船形开关 + 外露 USB-C 充电口

#### 硬件清单（总价约 ¥10）

| 元件 | 规格 | 价格 |
|------|------|------|
| 船形开关 | KCD11，3A/250V，带红色 LED 指示灯 | ¥3 |
| 短 USB-A 公头线 | 0.2 米，剪开接开关 | ¥3 |
| USB-C 母座面板式 | 9.5mm 面板安装 | ¥4 |

#### 接线示意

```
       充电宝（内置）
       ┌─────┐
       │ USB │ ←── 暴露在外壳上的 USB-C 充电口
       │ 出  │      (给充电宝充电用)
       └──┬──┘
          │ 短 USB-A 线（剪开）
          │
       ┌──┴──┐
       │ + ──→ 船形开关 (外露)
       │      └─→ 香橙派 5V Pin
       │ - ────────→ 香橙派 GND Pin
       └─────┘
```

#### 3D 外壳预留孔位

```
外壳侧面三孔布局：
┌────────────────────┐
│                    │
│  ⚪ ← 船形开关孔   │  21 × 15 mm（KCD11 标准卡扣）
│                    │
│  ⬛ ← USB-C 充电口 │  8 × 3 mm 长方孔
│                    │
│  • ← LED 指示孔    │  3 mm 圆孔（开关自带）
│                    │
└────────────────────┘
```

### 4.3 升级方案：UPS HAT 模块

| 优点 | 说明 |

## 五、充电宝选型（避坑指南）

### 5.1 致命问题：自动断电

**90% 的充电宝**在负载电流低于阈值（50-100mA）时会**自动关闭输出**，防止待机耗电。

后果：
- 系统进入低功耗瞬间，充电宝直接断电
- 开关切断后再打开，充电宝不会自动恢复供电

### 5.2 解决方案（三选一）

#### ① 选"持续输出"型充电宝

下单时搜索关键词：

```
"小电流充电宝"
"低功耗设备充电宝"
"涓流充电宝"
"树莓派/单板机 专用充电宝"
"Always On 充电宝"
```

已验证可用的型号（参考）：
- **倍思 30W PD 双向快充 10000mAh**
- **ALLPOWERS 单板机专用电源**
- **ROMOSS Sense 4 mini**（部分批次）
- **小米快充版 10000mAh**（部分批次）

#### ② 用 18650 电池组 + 升压模块（最彻底）

```
电池仓（4×18650 并联）→ MT3608 升压模块（升到 5V/3A）→ 开关 → 香橙派
```

总成本约 ¥30，完全没有自动断电问题，工程级方案。

#### ③ 用 UPS HAT（最专业）

直接买 Waveshare UPS HAT (B)，约 ¥120，专为单板机设计，零坑。

## 六、首次部署流程（一次性操作）

> ⚠️ 以下步骤需要通过 SSH 或键鼠登录到香橙派执行，**完成后现场就不需要这些操作了**。

### 6.1 准备阶段

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装依赖
sudo apt install -y python3-pip python3-venv git

# 3. 拉取项目
cd ~
git clone <your-repo> 26-Light
cd 26-Light/python

# 4. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 5. 安装 Python 依赖
pip install -r requirements.txt
pip install luma.lcd luma.core Pillow OPi.GPIO
```

### 6.2 开启硬件接口

```bash
sudo orangepi-config
# System → Hardware → 勾选：
#   ✓ spi-spidev    (小屏用)
#   ✓ uart0          (STM32 串口用)
# 保存 → 重启
```

### 6.3 配置 systemd 服务

按本文 **第二章** 操作。

### 6.4 配置桌面策略

按本文 **第三章** 选一种方案。

### 6.5 冷启动彩排（必做）

```
1. 拔掉香橙派所有电源、SSH、HDMI
2. 等 5 分钟（系统完全冷却）
3. 只插充电宝（或拨开关）
4. 计时观察：
   ✓ 30 秒内小屏亮起
   ✓ STM32 灯带显示对应等级
   ✓ 整个过程不碰任何键鼠
5. 至少彩排 3 次都成功，才算配置完成
```

## 七、现场演示完整流程

### 7.1 演示前一晚

- [ ] 把外壳充满电（插 USB-C 充几小时）
- [ ] 在家做一次完整冷启动测试
- [ ] 手机装好 Termius 并保存连接（应急用）
- [ ] 备好第二块充电宝（应急用）

### 7.2 演示现场

```
你的全部操作：
  ┌─ 拿出外壳
  ├─ 拨一下侧面船形开关
  ├─ 看到红色 LED 亮起
  ├─ 等 30 秒
  └─ 屏幕显示数据，开始演示

你的全部携带：
  ☑ 一个 3D 打印外壳（内含全部硬件）

```

> 严格来说应该先 SSH 进去 `sudo shutdown -h now` 优雅关机，但短时间直接断电不会损坏文件系统（ext4 有日志保护）。

## 八、应急方案（防止现场翻车）

### 8.1 应急工具组合

| 工具 | 用途 |
|------|------|
| 手机 + Termius | SSH 进香橙派排查 |
| 备用充电宝 | 主充电宝没电 |
| 短 USB-C 线 | 临时供电 |

### 8.2 常见故障对策

| 现象 | 排查方法 | 应急方案 |
|------|---------|---------|
| 拨开关后红灯不亮 | 充电宝没电 / 接触不良 | 拨回去再拨，或换充电宝 |
| 红灯亮但屏幕没显示 | systemd 服务没起来 | 拨回开关再拨（系统重启） |
| 屏幕亮但显示错误 | 程序崩溃中 | 等 5 秒看是否自动重启 |
| 等级一直不变 | 相机无法识别人脸 | 调整角度/光线 |
| 完全没反应 | 全挂了 | 拨回开关 30 秒再拨 |

### 8.3 SSH 应急连接（如果配置了 AP 模式）

```
1. 手机连香橙派 WiFi 热点（如 FatigueDemo / 12345678）
2. 打开 Termius
3. SSH 到 192.168.4.1
4. 常用命令：
   sudo systemctl status fatigue    # 查状态
   tail -f ~/fatigue.log            # 查日志
   sudo systemctl restart fatigue   # 重启程序
   sudo reboot                      # 重启系统
```

## 十、总结

### 10.1 核心三件事

1. **配置 systemd 服务** → 开机自动运行程序
2. **外露电源开关** → 不开盖即可启停
3. **选对充电宝** → 避免自动断电

### 10.2 一句话答复

> **「插电就能用吗？」**
>
> 是的。配好 systemd、装好外露开关、选对充电宝，
> 现场只需拨一下开关，30 秒后即可演示。
> 不需要电脑、不需要键鼠、不需要 SSH、不需要密码。
