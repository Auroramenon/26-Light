# STM32F103C8T6 灯带指令变色验证手册（ST-LINK + CH340G + SSCOM）

## 0. 目标
验证：电脑通过 USB-TTL（CH340G）发送串口指令后，STM32 能实时解析并控制 WS2812B 灯带按等级变色（并可同时观察蜂鸣器节奏变化）。

本项目当前固件逻辑：收到 `$FL,<level>,<hr>,<checksum>\r\n` 后，按 `level=0/1/2/3` 设置灯带颜色与蜂鸣器模式（3 级红灯闪烁）。

---

## 1. 硬件连接检查（务必先做）

### 1.1 ST-LINK（用于烧录/调试，走 SWD）
至少三线：
- ST-LINK SWDIO → STM32 SWDIO（通常是 PA13）
- ST-LINK SWCLK → STM32 SWCLK（通常是 PA14）
- ST-LINK GND → STM32 GND（必须共地）

建议再接：
- ST-LINK NRST → STM32 NRST（更容易“Connect under reset”）

> 提示：有些板子 SWD 引脚在独立的 4Pin/5Pin SWD 接口上，按丝印连接即可。

### 1.2 USB-TTL（CH340G，用于运行时发指令，走 USART1）
本固件使用 USART1（115200, 8N1），常见引脚：
- STM32 PA9  = USART1_TX
- STM32 PA10 = USART1_RX

交叉连接（重点）：
- CH340G TXD → STM32 PA10（RX）
- CH340G RXD → STM32 PA9（TX，可不接也能单向测试，但建议接）
- CH340G GND → STM32 GND（必须共地）

> 电平要求：CH340G 模块尽量使用 3.3V TTL 输出；若只能 5V TTL，长期不建议直连 PA10/PA9。

### 1.3 WS2812B 供电/地线（常见“能烧录但不工作”的根因）
- WS2812B 一般需要 5V 供电（电流要够）
- WS2812B 的 GND 必须与 STM32 GND 共地
- 若灯带较长/电流大，优先外接 5V 电源（不要指望 ST-LINK 供电带灯带）

## 2. 烧录固件（推荐：STM32CubeProgrammer + ST-LINK）

### 2.1 你现在工程里可用的固件文件
仓库已包含可烧录产物（ELF）：
- `stm32/Light/build/Debug/Light.elf`

> 如果你后续重新编译，产物可能会变化；但只要能得到 `.elf/.hex/.bin` 任意一种，都可以烧录。

### 2.2 安装工具
你已安装 STM32CubeMX，但 CubeMX 本身不负责烧录。

请确认已安装：
- STM32CubeProgrammer（ST 官方烧录工具）
- ST-LINK Driver（Windows 驱动）

> 如果你已能在设备管理器看到 ST-LINK 且 CubeProgrammer 能连接，一般驱动就 OK。

### 2.3 CubeProgrammer 烧录步骤
1. 打开 STM32CubeProgrammer
2. 左侧选择连接方式：ST-LINK
3. Interface 选择：SWD
4. 点击 Connect
   - 若连接失败，见本文“故障排查”
5. （建议首次）执行一次 Full chip erase（或 Erase）
6. 选择要下载的文件：`stm32/Light/build/Debug/Light.elf`
7. 点击 Download
8. 下载完成后点击 Run（或手动按板子复位键）

### 2.4 运行模式确认（BOOT0）
- 正常运行用户程序：BOOT0=0
- 若 BOOT0=1，芯片可能进系统 Bootloader，导致你以为“程序没跑起来”。

## 3. 用 SSCOM 发送指令验证变色

### 3.1 SSCOM 串口参数
1. 打开 SSCOM 5.13.1
2. 选择 CH340 对应 COM 口（设备管理器 → 端口 COM&LPT 可查看）
3. 串口参数设置：
   - 波特率：115200
   - 数据位：8
   - 校验：None
   - 停止位：1
4. 打开串口（Open）

### 3.2 发送格式要求（非常关键）
- 必须是 ASCII 文本帧
- 必须以 `\r\n` 结尾（回车换行）
- 校验必须正确，否则 STM32 会忽略该帧

协议格式：
```
$FL,<等级>,<心率>,<校验>\r\n
```

### 3.3 直接可用的测试指令（复制发送）
在 SSCOM 发送区粘贴后发送（确保会发送 CRLF；如软件支持“发送新行”，请选择 CRLF）：

```
$FL,0,72,3F\r\n
$FL,1,68,35\r\n
$FL,2,78,37\r\n
$FL,3,85,34\r\n
```

## 4. 预期现象（看到这些就说明链路 OK）
当你发送不同 level 后，灯带/蜂鸣器应该变为：
- level=0：绿灯常亮；蜂鸣器静音
- level=1：黄灯常亮；蜂鸣器慢间歇（约 1s 响 / 2s 停）
- level=2：橙灯常亮；蜂鸣器快间歇（约 300ms 响 / 300ms 停）
- level=3：红灯闪烁（约 500ms 周期）；蜂鸣器持续响

---

## 5. 如果测试指令没反应：最短排查路径

### 5.1 先排除“串口根本没接对”
- CH340G TXD 是否接到 STM32 的 RX（PA10）？
- 是否共地（CH340G GND 与 STM32 GND）？
- COM 口是否选对、串口是否已打开？
- 波特率是否 115200？

### 5.2 再排除“帧格式/结尾不对”
- SSCOM 是否真的发出了 `\r\n`？
  - 很多工具默认只发 `\n` 或不发换行；建议使用“发送新行=CRLF”。
- 校验是否正确？
  - 校验错：STM32 会当作无效帧丢弃（你看不到任何变化）。

### 5.3 再排除“供电/灯带问题”
- WS2812B 是否供电足够、且与 STM32 共地？
- 先只接 1 颗灯或短灯带验证（大电流会导致板子复位/异常）
- 若你使用 5V 灯带且数据线为 3.3V，部分灯带需要电平转换才能稳定识别（短距离有时也能跑，但不保证）。

## 6. （可选）自己计算 checksum，避免“手滑输错”
校验规则：对字符串 `FL,<level>,<hr>` 的每个字符做 XOR，得到 1 字节结果，再用两位十六进制表示。

你可以在电脑上用 Python 快速算（不改 STM32 代码）：
```python
def ck(level, hr):
    body = f"FL,{level},{hr}"
    x = 0
    for c in body:
        x ^= ord(c)
    return f"{x:02X}"

print(ck(0, 72))  # 期望输出 3F
```

## 8. 最快验证流程（建议照抄执行）
1. 用 ST-LINK 通过 CubeProgrammer 下载 `stm32/Light/build/Debug/Light.elf`
2. 确认 BOOT0=0，复位板子
3. 打开 SSCOM → 选 COM → 115200 8N1 → Open
4. 依次发送：
   - `$FL,0,72,3F\r\n`（看是否变绿）
   - `$FL,1,68,35\r\n`（看是否变黄+慢鸣）
   - `$FL,2,78,37\r\n`（看是否变橙+快鸣）
   - `$FL,3,85,34\r\n`（看是否红闪+持续鸣）

只要 0/1/2/3 任意一级能稳定触发，说明“串口→解析→反馈”全链路打通。
