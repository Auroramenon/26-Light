**发明名称：** 一种基于环境红外共模抑制与置信度自适应融合的夜间驾驶疲劳非接触预警方法及系统

**技术领域：** 智能交通安全 / 生物医学信号处理 / 计算机视觉

## 一、背景技术

### 1.1 现有技术

驾驶疲劳是道路交通事故的主要诱因之一。现有疲劳检测技术分为三类：

1. **接触式生理检测**：通过方向盘/座椅/穿戴设备采集心电、脉搏、皮肤电等。缺点是需要接触，影响驾驶体验，长期佩戴依从性差。
2. **可见光行为视觉检测**：通过摄像头识别闭眼（PERCLOS）、打哈欠、点头等行为。缺点是**夜间无可见光时完全失效**，而夜间恰是疲劳事故高发时段。
3. **可见光 rPPG（远程光电容积描记）非接触心率检测**：利用 R/G/B 三通道色度差分（如 CHROM、POS 算法）从面部提取脉搏波。其抑制光照干扰的原理依赖三个颜色通道，**在夜间无可见光、仅有近红外照明时无法工作**。

### 1.2 现有技术缺陷

将检测能力延伸到夜间的自然思路是采用 **850nm 近红外（NIR）主动照明 + NIR 相机**（850nm 光人眼不可见，不干扰驾驶）。但由此引出两个尚未被有效解决的技术难题：

**缺陷一（照明干扰）：** 近红外成像下每个像素只有单通道强度信息，无法使用 RGB rPPG 的色度差分去光照。心跳引起的皮肤反射率变化极其微弱（约 0.1%~0.5%），而夜间行车环境中，**对向车灯、路灯、隧道灯以及自车补光灯的抖动**会在 850nm 波段引入幅度高出脉搏信号 1~2 个数量级的共模强度波动，直接淹没脉搏信号，导致心率/HRV 检测失效。

**缺陷二（模态失效导致误报漏报）：** 多模态融合（心率+行为）通常采用**固定权重**加权。但真实行车中，各模态可靠性随时间剧烈波动：rPPG 在头部运动、照明突变时质量骤降；面部关键点在驾驶员回头、侧脸、遮挡时丢失。固定权重会把此刻失效模态的错误结论按原权重计入，造成**误报（虚警）或漏报**。

**缺陷三（报警疲劳与抖动）：** 简单阈值分级会在临界值附近频繁跳变、长时间持续鸣响，造成驾驶员对报警脱敏（报警疲劳），反而降低安全收益。

## 二、发明内容

### 2.1 发明目的

提供一种在**夜间**可用、**非接触**、**抗照明干扰**、**对模态失效鲁棒**、且**避免报警疲劳**的驾驶疲劳预警方法及系统。

### 2.2 技术方案总述

本发明的整体技术方案为一条闭环：

```
850nm主动照明 + NIR相机采集
        │
        ▼
人脸检测/跟踪 → 双区域采样（灌注区ROI + 非灌注参考区）
        │
        ▼
【要点一】环境红外共模抑制 → 干净脉搏波(BVP) → 心率/HRV
        │
        ├── 行为特征：PERCLOS / 哈欠 / 头姿
        │
        ▼
【要点二】信号质量指数(SQI)驱动的置信度自适应多模态融合 → 疲劳评分
        │
        ▼
【要点三】迟滞+持续性+多证据+自动恢复 分级状态机
        │
        ▼
分级光电闭环预警（WS2812B灯带颜色 + 蜂鸣器节律）
```

### 2.3 核心创新点

#### 要点一：车载近红外主动照明下的环境红外共模抑制（对应 `rppg/nir_illum.py`）

**要解决的技术问题：** 夜间近红外单通道成像中环境红外照明干扰淹没脉搏信号。

**技术方案：**

1. 在同一帧图像中**同步采样两个区域**：
   - **灌注区**：前额皮肤 ROI，同时包含脉搏信号（差模）与环境红外干扰（共模）；
   - **参考区**：人脸框左右两侧的非灌注背景条带，与灌注区受到**近似相同**的环境红外照明变化，但**几乎不含脉搏**。
2. 将两区域分别压缩为一维强度时间序列 `roi(t)` 与 `ref(t)`（灌注区采用额头中央权重更高的多子区加权，参考区取空间均值）。
3. 以**最小二乘**估计参考序列对灌注序列的最优共模增益 `a`：

   高通去基线后求解 `a = argmin_a ‖roi'(t) − a·ref'(t)‖²`，解析解为

   ```
   a = <roi', ref'> / <ref', ref'>       （对 a 限幅 [-3, 3] 防止病态参考过度对消）
   ```

4. 做**差分对消**：`signal(t) = roi(t) − a·ref(t)`，即从灌注信号中减去按最优增益缩放的共模照明分量，保留脉搏差模成分。
5. 再经去趋势、标准化、0.7~2.5Hz 带通滤波，得到干净脉搏波 BVP。
6. **鲁棒退化**：当参考区不可提取（人脸贴近画面边缘）时，自动退化为灌注区多子区加权强度法，保证系统始终可用。

**有益效果（已验证）：** 在叠加强共模照明干扰的合成数据上，共模抑制使脉搏波信号质量指数 SQI 由 0.31 提升至 0.67（提升约 1 倍），显著改善用于 HRV/逐拍间期分析的波形纯净度。该方案不依赖颜色通道，**填补了 RGB rPPG 无法用于夜间单通道近红外的空白**。

#### 要点二：基于信号质量指数（SQI）的置信度自适应多模态融合（对应 `rppg/signal_quality.py` + `fusion/feature_fusion.py`）

**要解决的技术问题：** 固定权重融合在模态瞬时失效时产生误报/漏报。

**技术方案：**

1. **实时评估各模态置信度**：
   - **HRV 通道**：定义信号质量指数 `SQI = 脉搏频带内主峰邻域能量 / 频带总能量`（谱聚集度，取值 [0,1]）。真实脉搏能量集中于心率频点附近，SQI 高；运动/照明干扰呈宽带，SQI 低。HRV 置信度由 `SQI` 与累积逐拍间期 `IBI` 数量共同决定（二者皆充分才可信）。
   - **行为通道**：置信度由人脸关键点可用性与头部运动幅度决定（关键点丢失或剧烈运动时降权）。
2. **动态权重再归一化**：以基础权重 `w_i` 乘以实时置信度 `conf_i` 后归一化：

   ```
   有效权重 w_i* = (w_i · conf_i) / Σ_j (w_j · conf_j)
   ```

   使不可信模态自动降权、可信模态自动补偿；当全部模态失效时退回基础权重，保证系统语义稳定。
3. 以有效权重对各模态风险分加权求和，再经 EMA 时间平滑得疲劳评分。

**有益效果（已验证）：** 当 HRV 通道信号质量骤降（SQI≈0.05）时，其权重由 0.20 自动降至 0.012，行为通道权重相应上升补偿，权重和恒为 1；反之当关键点丢失时 HRV 权重由 0.20 升至 0.556。有效抑制单一模态瞬时失效导致的虚警与漏报。

#### 要点三：迟滞—持续性—多证据—自动恢复的分级预警状态机（对应 `fusion/fatigue_classifier.py`）

**要解决的技术问题：** 阈值抖动与报警疲劳。

**技术方案：** 四级疲劳（正常/轻度/中度/重度）判定采用：
1. **升/降级双阈值迟滞**：升级阈值高于降级阈值，消除临界抖动；
2. **持续性约束**：目标等级需持续 `hold` 秒方生效，抑制瞬态误触发；
3. **Level 3 多证据要求**：判定重度疲劳须至少一路强证据（PERCLOS/HRV/哈欠达到高风险阈值），否则降为中度，避免最高级误报；
4. **快速恢复与自动恢复**：评分骤降时快速回落至正常；且各等级持续一定时长后自动恢复，避免长时间持续鸣响造成报警疲劳。

对应分级光电预警输出：

| 等级 | 名称 | WS2812B 灯带 | 蜂鸣器节律 |
|------|------|-------------|-----------|
| 0 | 正常 | 绿色常亮 | 静音 |
| 1 | 轻度 | 黄色常亮 | 1s 响 / 2s 停 |
| 2 | 中度 | 橙色常亮 | 300ms 响 / 300ms 停 |
| 3 | 重度 | 红色闪烁 | 持续报警 |

## 三、权利要求书（建议稿）

**1.** 一种夜间驾驶疲劳非接触预警方法，其特征在于，包括：
（a）以 850nm 近红外主动照明照射驾驶员面部，并以近红外相机采集面部图像序列；
（b）在每帧图像中同步确定**灌注区**与**非灌注参考区**，并分别提取灌注区强度时间序列与参考区强度时间序列；
（c）以最小二乘估计参考区序列对灌注区序列的共模增益，并从灌注区序列中减去按该增益缩放的参考区序列，得到抑制环境红外照明干扰后的脉搏波信号；
（d）由所述脉搏波信号估计心率及心率变异性（HRV）特征；
（e）由所述图像序列提取包括闭眼比例 PERCLOS、哈欠、头部姿态在内的行为特征；
（f）实时计算心率通道与行为通道各自的置信度，以该置信度对各通道基础融合权重加权并归一化，据以融合得到疲劳评分；
（g）依据所述疲劳评分经分级判定输出疲劳等级，并按等级驱动分级光电预警装置。

**2.** 根据权利要求 1 所述的方法，其特征在于，步骤（b）中所述参考区取人脸检测框左右两侧、与人脸中部竖直对齐的背景条带；当所述参考区不可提取时，步骤（c）退化为仅以灌注区多子区加权强度序列生成脉搏波信号。

**3.** 根据权利要求 1 所述的方法，其特征在于，步骤（c）中所述共模增益按解析式 `a = <roi', ref'> / <ref', ref'>` 求取，其中 `roi'`、`ref'` 为去除各自基线后的序列，且对 `a` 进行限幅以防过度对消。

**4.** 根据权利要求 1 所述的方法，其特征在于，步骤（f）中所述心率通道置信度基于信号质量指数 SQI 确定，所述 SQI 定义为脉搏频带内功率谱主峰邻域能量占该频带总能量的比例；且心率通道置信度进一步由累积逐拍间期数量共同确定。

**5.** 根据权利要求 1 所述的方法，其特征在于，步骤（f）中所述行为通道置信度由人脸关键点是否成功获取以及头部运动幅度确定；所述加权归一化为 `w_i* = w_i·conf_i / Σ_j w_j·conf_j`，且当各通道置信度均趋零时退回基础权重。

**6.** 根据权利要求 1 所述的方法，其特征在于，步骤（g）中所述分级判定包括：升级阈值与降级阈值不同的迟滞判定、目标等级须持续预定时长方生效的持续性约束、以及最高等级须满足至少一路强证据的多证据约束。

**7.** 根据权利要求 6 所述的方法，其特征在于，还包括自动恢复机制：任一非正常等级持续超过预定时长后自动恢复为正常等级，以避免持续报警造成的报警疲劳。

**8.** 根据权利要求 1 所述的方法，其特征在于，所述分级光电预警装置包括依疲劳等级切换颜色的 RGB 灯带与依疲劳等级切换节律的蜂鸣器，通过带校验的串行协议由上位机下发等级与心率驱动。

**9.** 根据权利要求 1 所述的方法，其特征在于，为适应嵌入式 ARM 平台算力，人脸关键点推理按每 N 帧执行一次、中间帧复用上一次关键点，且脉搏波与心率按固定间隔节流重算；并对近红外图像施加 CLAHE 自适应直方图均衡以提升关键点检测率。

**10.** 一种夜间驾驶疲劳非接触预警系统，包括 850nm 近红外补光阵列、850nm 滤光近红外相机、运行权利要求 1–9 任一所述方法的处理单元、以及由所述处理单元经串行接口驱动的 RGB 灯带与蜂鸣器分级预警装置。

## 四、具体实施方式（与代码对应）

### 4.1 硬件实施例
- 近红外相机（含 850nm 滤光片）+ 48 颗 850nm LED 补光阵列；
- 上位机（PC 或香橙派等 ARM 单板）运行 Python 检测程序；
- STM32F103 经 USB-TTL 接收等级/心率，驱动 WS2812B 灯带与有源蜂鸣器；
- 串行协议：`$FL,<level>,<hr>,<XOR校验>\r\n`（见 `docs/protocol.md`、`comm/protocol.py`）。

### 4.2 软件实施例（关键流程）
1. 采集与人脸检测/跟踪：`capture/camera.py`、`face/detector.py`、`face/tracker.py`。
2. 双区域提取：`face/roi.py` 的 `extract_roi`（灌注区）与 `extract_reference_region`（参考区）。
3. 共模抑制脉搏波：`rppg/nir_illum.py` 的 `extract_nir_bvp_cmr`、`estimate_common_mode_gain`。
4. 心率与 HRV：`rppg/hr_estimator.py`、`rppg/hrv_analyzer.py`。
5. 信号质量与置信度：`rppg/signal_quality.py` 的 `compute_bvp_sqi`、`hrv_confidence`、`behavior_confidence`。
6. 自适应融合：`fusion/feature_fusion.py` 的 `_adaptive_weights`、`FeatureFuser.fuse`。
7. 分级状态机：`fusion/fatigue_classifier.py` 的 `FatigueClassifier`。
8. 主循环编排：`python/main.py`。

### 4.3 参数实施例（`python/config.py`）
- `rppg_method="NIR_CMR"`、`illum_rejection_enabled=True`、`ref_strip_ratio=0.25`；
- `adaptive_fusion_enabled=True`、`sqi_band_low=0.7`、`sqi_band_high=2.5`；
- 基础权重 `w_hrv/w_perclos/w_yawn/w_head = 0.20/0.30/0.35/0.15`；
- 分级：`upgrade_thresholds=[0.35,0.55,0.70]`、`downgrade_thresholds=[0.25,0.45,0.60]`、`hold_seconds=3`。
