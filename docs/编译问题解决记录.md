# STM32 项目编译问题解决记录

## 问题描述

在 VS Code 中使用 CMake 编译 STM32 项目时遇到以下错误：

```
ninja: error: loading 'build.ninja': The system cannot find the file specified.
```

## 问题原因

1. **CMake 配置未成功**: build.ninja 文件没有生成，说明 CMake 配置阶段失败
2. **代码语法错误**: `fatigue_feedback.c` 第 28 行有多余的 `0` 字符
3. **重复编译**: CMakeLists.txt 中重复添加了已在 stm32cubemx 子目录中包含的源文件

## 解决步骤

### 1. 手动运行 CMake 配置

```bash
cd D:/guangdian/26-Light/stm32/Light
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
```

这一步会生成 build.ninja 文件。

### 2. 修复代码语法错误

**文件**: `Core/Src/fatigue_feedback.c`

**错误代码** (第 28 行):
```c
0    switch (level) {
```

**修复后**:
```c
    switch (level) {
```

### 3. 修复 CMakeLists.txt 重复编译问题

**文件**: `CMakeLists.txt`

**原始代码** (第 49-68 行):
```cmake
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    # User application sources
    Core/Src/main.c
    Core/Src/buzzer.c
    Core/Src/ws2812b.c
    Core/Src/uart_protocol.c
    Core/Src/fatigue_feedback.c
    Core/Src/tim.c
    Core/Src/usart.c
    Core/Src/gpio.c
    Core/Src/dma.c
    Core/Src/stm32f1xx_it.c
    Core/Src/stm32f1xx_hal_msp.c
    Core/Src/system_stm32f1xx.c
    Core/Src/syscalls.c
    Core/Src/sysmem.c

    # Startup file
    startup_stm32f103xb.s
)
```

**修复后**:
```cmake
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    # User application sources
    Core/Src/buzzer.c
    Core/Src/ws2812b.c
    Core/Src/uart_protocol.c
    Core/Src/fatigue_feedback.c

    # Note: Other sources (main.c, gpio.c, tim.c, usart.c, dma.c, etc.)
    # are already included by cmake/stm32cubemx/CMakeLists.txt
)
```

**原因**: `cmake/stm32cubemx/CMakeLists.txt` 已经包含了大部分源文件，只需要添加用户自定义的源文件（buzzer.c, ws2812b.c, uart_protocol.c, fatigue_feedback.c）。

### 4. 重新编译

```bash
cd D:/guangdian/26-Light/stm32/Light/build
ninja
```

## 在 VS Code 中使用

### 方法 1: 使用 CMake 扩展

1. 打开命令面板 (Ctrl+Shift+P)
2. 选择 "CMake: Configure"
3. 选择 "CMake: Build"

### 方法 2: 使用终端

```bash
# 配置
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug

# 编译
cmake --build build

# 或直接使用 ninja
cd build && ninja
```

### 方法 3: 使用 Tasks

在 `.vscode/tasks.json` 中配置任务：

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "STM32: Configure",
            "type": "shell",
            "command": "cmake",
            "args": [
                "-B", "build",
                "-G", "Ninja",
                "-DCMAKE_BUILD_TYPE=Debug"
            ],
            "options": {
                "cwd": "${workspaceFolder}/stm32/Light"
            }
        },
        {
            "label": "STM32: Build",
            "type": "shell",
            "command": "ninja",
            "options": {
                "cwd": "${workspaceFolder}/stm32/Light/build"
            },
            "dependsOn": ["STM32: Configure"],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}
```

然后使用 Ctrl+Shift+B 快速编译。

## 常见问题

### Q1: 找不到 arm-none-eabi-gcc

**解决**: 确保 ARM 工具链在 PATH 中

```bash
# 检查
where arm-none-eabi-gcc

# 如果没有，添加到 PATH:
# C:\Program Files (x86)\Arm GNU Toolchain arm-none-eabi\14.2 rel1\bin
```

### Q2: ninja: error: loading 'build.ninja'

**解决**: 先运行 CMake 配置

```bash
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
```

### Q3: multiple definition of 'xxx'

**解决**: 检查 CMakeLists.txt 是否重复添加了源文件。只添加用户自定义的源文件，STM32CubeMX 生成的文件已经在 `cmake/stm32cubemx/CMakeLists.txt` 中了。

### Q4: 编译很慢

**解决**: 使用 Ninja 而不是 Make

```bash
cmake -B build -G Ninja  # 使用 Ninja (快)
cmake -B build -G "Unix Makefiles"  # 使用 Make (慢)
```
