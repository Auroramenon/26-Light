# 疲劳驾驶光电预警系统 — 使用指南

## 一、系统总览

本系统由两部分组成：

- PC 端 Python 程序：通过工业相机采集面部视频，运行 rPPG 算法提取心率/HRV，结合行为特征（PERCLOS、哈欠、头姿）进行多模态融合，输出四级疲劳等级。
- STM32 端固件：接收 PC 端的疲劳等级信号，驱动 WS2812B LED 灯带变色、蜂鸣器报警。

两者通过 USB-to-TTL 串口模块连接，协议为 ASCII 文本帧。

## 四、各元件杜邦线连接详解

### 4.1 USB-to-TTL 模块 ↔ STM32 (串口通信)

USB-to-TTL 模块一端插电脑 USB 口，另一端通过杜邦线连接 STM32。

```
USB-to-TTL 模块          STM32 Blue Pill
┌─────────────┐          ┌──────────────┐
│ TXD (发送)  │───白线──→│ PA10 (RX)    │
│ RXD (接收)  │───绿线──→│ PA9  (TX)    │
│ GND (地线)  │───黑线──→│ GND          │
│ VCC         │    不接   │              │
└─────────────┘          └──────────────┘
```

连接要点：

- TX 接 RX，RX 接 TX（交叉连接）
- GND 必须连接（共地），否则通信不稳定
- VCC 不要连接到 STM32 的 3.3V，STM32 通过自身 Micro-USB 供电
- 如果 USB-to-TTL 模块是 5V 电平的（如某些 CH340 模块），需确认 STM32 的 PA10 是否 5V 容忍（F103 的大部分引脚是 5V 容忍的，PA10 可以）

### 4.2 WS2812B LED 灯带 ↔ STM32

WS2812B 是单线协议，只需要一根数据线。

```
WS2812B 灯带              STM32 Blue Pill
┌──────────────┐          ┌──────────────┐
│ DIN (数据入) │───黄线──→│ PA0          │
│ VCC (+5V)    │───红线──→│ 5V           │
│ GND (地线)   │───黑线──→│ GND          │
└──────────────┘          └──────────────┘

灯带方向:  DIN ●━━●━━●━━●━━●━━●━━●━━● DOUT
           第1颗                    第8颗
```

连接要点：

- DIN 是灯带的数据输入端，注意灯带有方向（箭头指向数据流方向）
- 必须连接到灯带的 DIN 端，不是 DOUT 端
- VCC 接 5V 电源。如果灯珠较多（>8颗），建议使用独立 5V 电源，不要从 STM32 的 5V 引脚取电（电流不够）
- 建议在 DIN 线上串联一个 330Ω 电阻（保护第一颗灯珠）
- 建议在 VCC 和 GND 之间并联一个 100μF 电解电容（滤波稳压）

```
推荐电路:

PA0 ──[330Ω]──→ DIN
                  │
5V  ──────────→ VCC ──┤├── GND    (100μF 电解电容)
                  │         │
GND ──────────→ GND ───────┘
```

### 4.3 有源蜂鸣器模块 ↔ STM32

有源蜂鸣器模块通常有 3 个引脚：VCC、GND、I/O（信号）。

```
蜂鸣器模块                STM32 Blue Pill
┌──────────────┐          ┌──────────────┐
│ I/O (信号)   │───蓝线──→│ PA6          │
│ VCC (+5V)    │───红线──→│ 5V           │
│ GND (地线)   │───黑线──→│ GND          │
└──────────────┘          └──────────────┘
```

连接要点：

- I/O 引脚接高电平时蜂鸣器发声，低电平时静音
- 程序通过 PWM 控制 I/O 引脚的高低电平切换，实现间歇报警效果
- 如果蜂鸣器模块自带三极管驱动电路（大多数模块都有），可以直接连接
- 如果是裸蜂鸣器（无驱动板），需要外接一个 NPN 三极管（如 S8050）驱动

### 4.4 工业相机 ↔ 电脑

```
LRCP S400 工业相机
┌──────────────┐          ┌──────────────┐
│ USB 3.0 接口 │══USB线══→│ 电脑 USB 3.0 │
└──────────────┘          └──────────────┘
```

- 直接通过 USB 3.0 数据线连接电脑
- 确保插在电脑的 USB 3.0 接口上（蓝色接口），以保证带宽
- 首次使用可能需要安装相机驱动

### 4.5 完整接线总图

```
                    ┌─────────────────────────────────────┐
                    │              电脑 (PC)               │
                    │                                     │
                    │  USB 3.0 口 ←── LRCP S400 相机      │
                    │  USB 口    ←── USB-to-TTL 模块      │
                    │  USB 口    ←── STM32 Micro-USB      │
                    └─────────────────────────────────────┘
                              │
                         USB-to-TTL
                         TXD ──→ PA10
                         RXD ──→ PA9
                         GND ──→ GND
                              │
                    ┌─────────┴─────────┐

                    │                   │
                    │  PA0 ──→ WS2812B DIN
                    │  PA6 ──→ 蜂鸣器 I/O
                    │  5V  ──→ WS2812B VCC + 蜂鸣器 VCC
                    │  GND ──→ 公共地线
                    └───────────────────┘
                              │
              ┌───────────────┼───────────────┐
              │               │               │
        ┌─────┴─────┐  ┌─────┴─────┐  ┌──────┴──────┐
        │ WS2812B   │  │  蜂鸣器   │  │ 850nm LED   │
```

## 五、供电方案

### 方案 A：USB 统一供电（推荐调试阶段）

- STM32 通过 Micro-USB 连接电脑供电（5V/500mA）
- WS2812B 和蜂鸣器从 STM32 的 5V 引脚取电
- 适用于 LED 数量 ≤ 8 颗的情况

### 方案 B：独立电源供电（推荐正式部署）

- STM32 通过 Micro-USB 供电
- WS2812B 灯带使用独立 5V/2A 电源供电
- 所有模块的 GND 必须连在一起（共地）

```
独立供电接线:

5V/2A 电源 ──→ WS2812B VCC
             ──→ 蜂鸣器 VCC
             ──→ GND (与 STM32 GND 连接)
```

## 六、串口使用说明

### 6.1 确认串口号

Windows 系统：

1. 将 USB-to-TTL 模块插入电脑
2. 打开「设备管理器」→「端口 (COM 和 LPT)」
3. 找到类似 `USB-SERIAL CH340 (COM3)` 的条目
4. 记下 COM 号（如 COM3）

如果看不到设备，需要安装对应芯片的驱动：

- CH340：搜索「CH340 驱动」下载安装
- CP2102：搜索「CP2102 驱动」下载安装
- FT232：搜索「FTDI 驱动」下载安装

### 6.2 串口参数

| 参数   | 值        |
| ------ | --------- |
| 波特率 | 115200    |
| 数据位 | 8         |
| 校验位 | 无 (None) |
| 停止位 | 1         |
| 流控制 | 无        |

### 6.3 串口调试

可以使用串口调试助手（如 SSCOM、PuTTY、或 VSCode 的 Serial Monitor）手动测试 STM32：

```
发送以下文本测试各等级响应:

$FL,0,72,3F\r\n    -- 正常, 心率72
$FL,1,68,35\r\n    -- 轻度疲劳, 心率68
$FL,2,78,37\r\n    -- 中度疲劳, 心率78
$FL,3,85,34\r\n    -- 重度疲劳, 心率85
```

注意：发送时需要勾选「发送新行」或手动添加 `\r\n` 结尾。

### 6.4 协议格式

```
$FL,<等级>,<心率>,<校验>\r\n

字段说明:
  $FL      — 固定包头
  等级     — 0/1/2/3
  心率     — 整数 BPM (40-200)
  校验     — "FL,<等级>,<心率>" 所有字符 XOR 后的 2 位十六进制
  \r\n     — 回车换行结尾
```

Python 端校验计算示例：

```python
body = f"FL,{level},{hr}"
checksum = 0
for c in body:
    checksum ^= ord(c)
packet = f"${body},{checksum:02X}\r\n"
```

### 7.2 运行 Python 程序

基本用法：

```bash
# 仅使用相机测试（不连接串口）
python main.py --no-serial

# 指定相机编号
python main.py --camera 1 --no-serial

# 连接串口
python main.py --serial COM8

# 使用 POS 算法替代默认的 CHROM
python main.py --method POS --serial COM8

# 使用 YOLO5Face 人脸检测（需要 GPU）
python main.py --face-backend Y5F --serial COM8

# 无 GUI 模式（纯后台运行）
python main.py --no-gui --serial COM8
```

命令行参数一览：

| 参数               | 说明                    | 默认值 |
| ------------------ | ----------------------- | ------ |
| `--camera`       | 相机设备索引            | 0      |
| `--serial`       | 串口号                  | COM8   |
| `--method`       | rPPG 算法 (CHROM / POS) | CHROM  |
| `--face-backend` | 人脸检测 (HC / Y5F)     | HC     |
| `--no-serial`    | 禁用串口输出            | —     |
| `--no-gui`       | 禁用 GUI 窗口           | —     |

## 八、疲劳等级与反馈对照表

| 等级 | 名称     | 综合评分  | LED 灯带         | 蜂鸣器            |
| ---- | -------- | --------- | ---------------- | ----------------- |
| 0    | 正常     | < 0.3     | 绿色常亮         | 静音              |
| 1    | 轻度疲劳 | 0.3 ~ 0.5 | 黄色常亮         | 1秒响 / 2秒停     |
| 2    | 中度疲劳 | 0.5 ~ 0.7 | 橙色常亮         | 300ms响 / 300ms停 |
| 3    | 重度疲劳 | ≥ 0.7    | 红色闪烁 (500ms) | 持续报警          |

综合评分 = 0.35×HRV评分 + 0.30×PERCLOS + 0.20×哈欠频率 + 0.15×头部俯仰

## 九、常见问题排查

### 串口连不上

1. 检查设备管理器中是否识别到 COM 口
2. 确认 TX/RX 是否交叉连接（不是同名对接）
3. 确认 GND 是否连接
4. 确认波特率设置为 115200
5. 尝试更换 USB-to-TTL 模块或 USB 线

### WS2812B 灯带不亮

1. 检查 DIN 方向是否正确（灯带有箭头标识）
2. 检查 5V 供电是否正常（用万用表测量）
3. 检查 GND 是否与 STM32 共地
4. 尝试减少灯珠数量测试（先测 1 颗）
5. 检查 PA0 引脚是否正确连接

### 蜂鸣器不响

1. 确认是有源蜂鸣器（自带振荡电路，通电即响）
2. 用杜邦线直接将蜂鸣器 I/O 接 5V 测试是否能响
3. 检查 PA6 引脚连接
4. 确认蜂鸣器模块的 VCC 接了 5V

### 相机打不开

1. 确认相机 USB 线插在 USB 3.0 接口上
2. 尝试修改 `--camera` 参数（0, 1, 2 逐个试）
3. 检查是否安装了相机驱动
4. 用其他软件（如 AMCap）测试相机是否正常

### rPPG 心率不准

1. 确保面部光照均匀，避免强逆光
2. 保持头部相对静止（运动伪影会干扰信号）
3. 等待至少 8 秒让信号缓冲区填满
4. 尝试切换算法：`--method POS`
5. 如果使用近红外光源，确保 850nm LED 正常工作

## 十、配置文件调参

所有可调参数集中在 `python/config.py` 中：

```python
# 调整 rPPG 信号窗口（越长越稳定，但延迟越大）
"signal_window_sec": 8,        # 建议 6-10 秒

# 调整闭眼检测灵敏度
"ear_threshold": 0.2,          # 越大越容易判定为闭眼

# 调整哈欠检测灵敏度
"yawn_mar_threshold": 0.6,     # 越小越容易触发

# 调整疲劳分级阈值
"fatigue_thresholds": [0.3, 0.5, 0.7],  # 可降低使系统更敏感

# 调整各模态权重
"w_hrv": 0.35,                 # HRV 生理指标权重
"w_perclos": 0.30,             # 闭眼比例权重
"w_yawn": 0.20,                # 哈欠频率权重
"w_head": 0.15,                # 头部姿态权重
```
