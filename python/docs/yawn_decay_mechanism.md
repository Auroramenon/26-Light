# 哈欠衰减机制说明

## 问题描述

原有的哈欠检测使用60秒滑动窗口统计，导致：
- 打1次哈欠后，接下来60秒内哈欠率都显示为1次/分
- 即使驾驶员已经清醒，系统仍会因为"历史记录"反复触发警报
- 结合自动恢复机制，会出现：响铃5秒 → 停止 → 5秒后再响 → 循环

## 解决方案：哈欠衰减机制

### 工作原理

哈欠记录不再是简单的计数，而是带有**时间衰减权重**：

```
0-30秒：权重 = 1.0（完全计入）
30-60秒：权重线性衰减 1.0 → 0.0
60秒后：完全移除
```

### 实际效果

**场景：驾驶员打了1次哈欠**

| 时间 | 原有机制 | 新机制（衰减） | 说明 |
|------|---------|---------------|------|
| 0秒 | 1次/分 | 1.0次/分 | 刚打完哈欠 |
| 15秒 | 1次/分 | 1.0次/分 | 仍在30秒窗口内 |
| 30秒 | 1次/分 | 1.0次/分 | 开始衰减 |
| 45秒 | 1次/分 | 0.5次/分 | 衰减到一半 |
| 55秒 | 1次/分 | 0.17次/分 | 接近消失 |
| 60秒 | 0次/分 | 0次/分 | 完全移除 |

### 与风险映射的配合

配置中的哈欠风险阈值：
- `yawn_rate_low: 0.2` - 低于0.2次/分，风险为0
- `yawn_rate_high: 1.0` - 高于1.0次/分，风险为1

**效果**：
- 0-30秒：哈欠率1.0，风险为1.0（高风险）
- 45秒：哈欠率0.5，风险约0.375（中等风险）
- 55秒：哈欠率0.17，风险为0（低于阈值）

这样，打1次哈欠后约**50-55秒**就不会再触发警报了。

## 配置参数

在 `config.py` 中：

```python
"yawn_window_sec": 60.0,        # 统计窗口（秒）
"yawn_decay_enabled": True,     # 启用衰减机制
"yawn_decay_window": 30.0,      # 30秒后开始衰减
```

### 调整建议

**如果觉得衰减太慢**（仍然反复响铃）：
```python
"yawn_decay_window": 20.0,      # 20秒后就开始衰减
```

**如果觉得衰减太快**（哈欠警报不够持久）：
```python
"yawn_decay_window": 40.0,      # 40秒后才开始衰减
```

**如果不想要衰减**（保持原有行为）：
```python
"yawn_decay_enabled": False,    # 禁用衰减
```

## 完整流程示例

### 场景：驾驶员打哈欠后清醒

```
时间    动作              哈欠率    疲劳评分    等级    说明
0s      打哈欠            1.0       0.40        1       触发Level 1，响铃
5s      已清醒            1.0       0.38        1       自动恢复到Level 0，停止响铃
10s     保持清醒          1.0       0.35        0       评分下降，但哈欠率仍高
15s     保持清醒          1.0       0.32        0       不会再触发（评分低于0.35）
30s     保持清醒          1.0       0.30        0       开始衰减
45s     保持清醒          0.5       0.28        0       哈欠率衰减到0.5
55s     保持清醒          0.17      0.25        0       哈欠率低于阈值，完全恢复
```

**关键点**：
- 自动恢复机制：5秒后停止响铃
- 哈欠衰减：45秒后哈欠率降低，不再触发新警报
- 评分下降：驾驶员清醒后，综合评分也会下降

### 场景：驾驶员持续疲劳

```
时间    动作              哈欠率    疲劳评分    等级    说明
0s      打哈欠            1.0       0.40        1       触发Level 1，响铃
5s      仍疲劳            1.0       0.42        1       自动恢复到Level 0
10s     又打哈欠          2.0       0.45        1       再次触发Level 1（哈欠率增加）
15s     仍疲劳            2.0       0.48        1       自动恢复到Level 0
20s     继续疲劳          2.0       0.50        2       触发Level 2（评分升高）
```

**关键点**：
- 如果持续疲劳，会反复触发警报
- 多次哈欠会累积，哈欠率增加
- 系统不会遗漏真正的疲劳状态

## 与其他机制的配合

系统现在有多层保护机制：

1. **哈欠衰减**：历史哈欠记录随时间衰减
2. **自动恢复**：警报持续5-7秒后自动停止
3. **快速恢复**：评分低于0.20立即恢复
4. **迟滞阈值**：升降级有不同阈值，防止抖动

这些机制共同作用，确保：
- ✅ 真正疲劳时会及时警报
- ✅ 警报后不会长时间响铃
- ✅ 清醒后不会反复误报
- ✅ 持续疲劳会反复提醒
