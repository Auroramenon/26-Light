# 疲劳检测参数配置说明

## 核心改进

### 1. 更符合现实场景的阈值设置

**风险映射参数**（保持原有合理值）：
- PERCLOS: 0.15-0.30（15%-30%闭眼比例）
- 哈欠率: 0.2-1.0次/分
- 头姿偏转: 15°-35°

这些阈值基于实际驾驶疲劳研究，不会过于敏感导致误报。

### 2. 解决降级困难问题

**关键改进**：
- **升级阈值**: [0.35, 0.55, 0.75] - 需要明确的疲劳信号才升级
- **降级阈值**: [0.25, 0.45, 0.65] - 差距0.10，更容易降级
- **快速恢复机制**: 当评分低于0.20时，直接降至Level 0

**为什么这样设计**：
```
Level 0 → Level 1: 需要评分 >= 0.35
Level 1 → Level 0: 只需评分 < 0.25

差距0.10，给予足够的降级空间
```

### 3. 自动恢复机制（防止长时间响铃）

**新增功能**：疲劳等级持续一定时间后自动恢复到Level 0

- **Level 1**: 持续5秒后自动恢复
- **Level 2**: 持续5秒后自动恢复
- **Level 3**: 持续7秒后自动恢复

**设计目的**：
- 响铃提醒驾驶员后自动复位
- 避免长时间响铃造成干扰
- 如果驾驶员仍然疲劳，会再次触发警报

### 4. 快速恢复机制

当疲劳评分低于0.20（很低）时：
- 立即降至Level 0，无需等待持续时间
- 适用场景：驾驶员休息后恢复、误判后快速纠正

## 参数详解

### 疲劳等级判定流程

```
评分范围        等级        说明
0.00-0.25      Level 0     正常状态
0.25-0.35      过渡区      可能从Level 1降级
0.35-0.45      Level 1     轻度疲劳
0.45-0.55      过渡区      Level 1/2之间
0.55-0.65      Level 2     中度疲劳
0.65-0.75      过渡区      Level 2/3之间
0.75-1.00      Level 3     重度疲劳（需多证据）
```

### 迟滞阈值设计

**升级（需要更强证据）**：
- 0 → 1: score >= 0.35
- 1 → 2: score >= 0.55
- 2 → 3: score >= 0.75

**降级（更容易恢复）**：
- 1 → 0: score < 0.25
- 2 → 1: score < 0.45
- 3 → 2: score < 0.65

**快速恢复**：
- 任何等级 → 0: score < 0.20（立即生效）

### Level 3 多证据要求

重度疲劳需要满足以下之一：
1. PERCLOS风险 >= 0.90（闭眼时间占比很高）
2. HRV风险 >= 0.80（心率变异性显著异常）
3. 哈欠风险 >= 0.70 且 头姿风险 >= 0.60（组合指标）

这些阈值较高，避免误判重度疲劳。

## 实际场景示例

### 场景1：正常驾驶 → 轻度疲劳 → 自动恢复

```
时间    评分    等级    说明
0s      0.20    0      正常驾驶
30s     0.38    1      开始疲劳（持续3秒后升级到Level 1）
33s     0.40    1      Level 1，开始响铃
38s     0.42    1      Level 1持续5秒，自动恢复到Level 0，停止响铃
40s     0.35    0      保持Level 0
```

### 场景2：中度疲劳警报

```
时间    评分    等级    说明
0s      0.45    1      轻度疲劳
10s     0.58    2      升级到Level 2（中度疲劳），开始响铃
15s     0.60    2      Level 2持续5秒，自动恢复到Level 0，停止响铃
20s     0.55    0      保持Level 0（如果仍疲劳会再次触发）
```

### 场景3：重度疲劳警报

```
时间    评分    等级    说明
0s      0.70    2      中度疲劳
10s     0.78    3      升级到Level 3（重度疲劳），强烈响铃
17s     0.80    3      Level 3持续7秒，自动恢复到Level 0，停止响铃
20s     0.75    0      保持Level 0（如果仍疲劳会再次触发）
```

### 场景4：快速恢复（驾驶员调整状态）

```
时间    评分    等级    说明
0s      0.45    1      轻度疲劳，响铃
3s      0.18    0      驾驶员调整状态，评分骤降至0.20以下，立即恢复到Level 0
```

### 场景5：持续疲劳（反复触发）

```
时间    评分    等级    说明
0s      0.40    1      轻度疲劳，响铃
5s      0.42    1      持续5秒，自动恢复到Level 0
8s      0.40    1      评分仍高，再次触发Level 1，再次响铃
13s     0.38    1      持续5秒，再次自动恢复到Level 0
```

**说明**：如果驾驶员持续疲劳，系统会反复提醒，但每次提醒后会自动复位，避免长时间响铃。

## 调优建议

### 如果觉得升级太快
- 提高 `upgrade_thresholds`（如 [0.40, 0.60, 0.80]）
- 增加 `hold_seconds`（如 5秒）

### 如果觉得降级太慢
- 降低 `downgrade_thresholds`（如 [0.20, 0.40, 0.60]）
- 降低 `recovery_threshold`（如 0.15）

### 如果觉得响铃时间太短
- 增加 `auto_reset_durations` 中的时间：
  ```python
  "auto_reset_durations": {
      1: 8.0,   # Level 1 持续8秒
      2: 10.0,  # Level 2 持续10秒
      3: 12.0   # Level 3 持续12秒
  }
  ```

### 如果觉得响铃时间太长
- 减少 `auto_reset_durations` 中的时间：
  ```python
  "auto_reset_durations": {
      1: 3.0,   # Level 1 持续3秒
      2: 4.0,   # Level 2 持续4秒
      3: 5.0    # Level 3 持续5秒
  }
  ```

### 如果不想要自动恢复功能
- 设置 `auto_reset_enabled: False`
- 系统将保持原有的降级逻辑（需要评分降低才会降级）

### 如果Level 3误报太多
- 提高 `level3_min_*_risk` 阈值
- 或设置 `level3_multi_evidence: False` 禁用多证据要求

### 如果想要更敏感的检测
- 降低 `upgrade_thresholds`
- 提高各路风险权重中的 `w_perclos`

## 自动恢复机制详解

### 工作原理

1. **触发警报**：当疲劳评分达到阈值，系统升级到相应等级并开始响铃
2. **计时开始**：系统记录进入该等级的时间
3. **持续响铃**：在设定的时间内持续响铃提醒
4. **自动复位**：时间到达后自动恢复到Level 0，停止响铃
5. **再次检测**：如果驾驶员仍然疲劳，会再次触发警报

### 优势

- ✅ 避免长时间响铃造成干扰
- ✅ 提醒后自动复位，等待下次检测
- ✅ 持续疲劳会反复提醒，不会遗漏
- ✅ 响铃时间可根据等级严重程度调整

### 与其他恢复机制的关系

系统有三种恢复机制，按优先级排序：

1. **自动恢复**（最高优先级）：等级持续一定时间后自动恢复
2. **快速恢复**：评分低于0.20时立即恢复
3. **正常降级**：评分降至降级阈值以下时降级

这三种机制互不冲突，共同确保系统能够及时恢复到正常状态。
