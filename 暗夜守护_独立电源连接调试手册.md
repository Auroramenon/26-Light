# 暗夜守护系统 — 香橙派 5 独立电源供电 · 连接与调试手册

> 目标：让香橙派 5 + 近红外摄像头 + STM32 预警模块脱离 220V 插座，依靠便携电源稳定运行，运行检测程序时不再自动断电。

---

## 一、器材清单与角色

| 器材 | 你买的规格 | 在系统里的角色 |
|------|-----------|---------------|
| C-C 数据线 | Type-C 双头，0.5m，100W，E-Marker | 给香橙派供电的主干线（低线损、抗大电流） |
| PD 诱骗器 | Type-C 公头输入，Type-C / USB 输出 | 强制电源输出 **5V**，绕开 PD 协商失败的风险 |
| Z-TEK 力特 USB Hub | 5V/2A 有源，DC 供电口，0.5m 上联线 | 给摄像头**独立供电**，把摄像头电流从香橙派电源链路剥离 |
| USB 转 DC 5V 电源线 | USB-A 输入 → DC 输出 | 用充电器/诱骗器的 USB 口给 Hub 的 DC 口供电 |
| 大功率充电器 | PD 多口（假设含 Type-C + USB-A） | 系统总电源 |

> ⚠️ **关于"独立运行"的提醒**：`充电器`需要插 220V 插座，**不是移动电源**。
> - 如果是**桌面/有插座的场地**演示 → 用充电器即可。
> - 如果要**塞进车里真正独立运行** → 把"大功率充电器"换成**支持 PD 的多口移动电源（充电宝）**，电气接法完全一样。下文统一称为"PD 电源"。

---

## 二、为什么这样接（核心原理）

你最初断电的根本原因：**普通 USB-A 口只有 5V/2A，程序启动瞬间 CPU 满载 + 摄像头初始化，峰值电流冲到 ~2.5A，超过电源的过流保护阈值，啪——断电。**

这套器材从两个方向解决问题：

1. **抬高香橙派供电能力**：PD 5V/3A + 0.5m 100W 线，把上限从 2A 抬到 3A，并大幅降低线压降。
2. **剥离摄像头负载**：摄像头改由有源 Hub 独立供电，香橙派电源链路只剩 CPU 自己的电流，峰值不再叠加。

```
香橙派满载        ~1.5A
启动峰值          ~2.5A
─────────────────────────
PD 5V/3A 上限     3.0A   ← 摄像头剥离后留 ~0.5A 余量 ✅
```

## 三、接线方案

### 方案一：充电器双口分供（推荐，最稳）

> 适用：大功率充电器同时有 **Type-C 口** 和 **USB-A 口**。两路电源各自独立，互不抢电。

```
                       ┌──────────────────────────────┐
                       │        大功率充电器 / PD电源     │
                       └──────────┬──────────┬──────────┘
                          Type-C口 │          │ USB-A口
                                   │          │
                    C-C线(0.5m,100W)│          │ USB转DC5V线
                                   │          │
                                   ▼          ▼
                       ┌────────────────┐  ┌──────────────────┐
                       │   香橙派 5      │  │  Z-TEK Hub DC口  │
                       │                │  │                  │
                       │  Type-C供电口◀─┘  │  下游口 ──────────┼──▶ 近红外摄像头
                       │                   │                  │
                       │  USB-A口 ◀────────┼── Hub上联线(0.5m) │
                       │                   └──────────────────┘
                       │  HDMI口 ──▶ 显示屏 │
                       └───────────────────┘
```

**连接步骤：**

1. **香橙派供电**：充电器 `Type-C 口` → `C-C 100W 线` → 香橙派 `Type-C 供电口`。
2. **Hub 供电**：充电器 `USB-A 口` → `USB 转 DC 5V 线` → Z-TEK Hub 的 `DC 供电口`。
3. **数据上联**：Z-TEK Hub 的 `上联线(0.5m)` → 香橙派任意 `USB-A 口`（优先黑色 USB 2.0 口）。
4. **摄像头**：摄像头 USB 线 → Hub 任意 `下游口`。**绝不直接插香橙派。**
5. **显示**：香橙派 `HDMI 口` → 显示屏。

> PD 诱骗器在本方案中**暂时不用**，留作"方案三"的保险件。

---

### 方案二：PD 诱骗器单源分供（接线少，但电流临界）

> 适用：充电器**只有 Type-C 口**，没有 USB-A 口；或你想只用一个输出口。

```
        大功率充电器 [Type-C口]
                 ▲
                 │ PD诱骗器(Type-C公头) 直接插入
                 │
        ┌────────┴─────────┐
        │   PD 诱骗器       │  请求并锁定 5V
        │                  │
        │ [Type-C输出]─────┼─ C-C线 ─▶ 香橙派 Type-C供电口
        │                  │
        │ [USB输出]────────┼─ USB转DC线 ─▶ Z-TEK Hub DC口 ─▶ 摄像头
        └──────────────────┘
        香橙派 USB-A口 ◀── Hub上联线
```

> ⚠️ **重要限制**：PD 协议 **5V 档最高 3A**。此方案下香橙派(~2.5A) + Hub/摄像头(~0.5A) = **3A，正好压在天花板上**，峰值时可能仍会触发保护。
> **能用方案一就别用方案二。** 方案二仅在充电器接口不够时作为备选。

---

### 方案三：诱骗器作"5V 保险"（方案一的补丁）

如果**方案一中香橙派直连充电器后协商失败**（表现：开不了机、刚启动就掉电、提示欠压），把 PD 诱骗器插进香橙派那一路，强制锁定 5V：

```
充电器 Type-C口 ◀── PD诱骗器(公头插入) → [Type-C输出] → C-C线 → 香橙派
```

诱骗器替香橙派完成 PD 协商，输出稳定 5V，绕开兼容性问题。

## 四、上电顺序

> 顺序错了可能导致摄像头枚举异常，按此操作最稳：

1. 接好**全部线缆**（此时电源还没开）。
2. **先给 Hub 上电**（充电器 USB-A 口已通电 / 或先确认 Hub 指示灯亮）。
3. **再给香橙派上电**（PD 那一路通电）。
4. 等待香橙派启动（约 30 秒），显示屏出画面。
5. 启动后再运行检测程序。

## 五、系统识别验证

香橙派启动后，打开终端依次执行，确认硬件被正确识别。

**1. 查看 USB 设备（应同时看到 Hub 和摄像头）**

```bash
lsusb
```

期望输出里能找到两类设备：

```
Bus 00x Device 00x: ID xxxx:xxxx  ... Hub
Bus 00x Device 00x: ID xxxx:xxxx  ... Camera / Webcam / UVC
```

**2. 查看摄像头设备节点**

```bash
ls /dev/video*
```

期望出现 `/dev/video0`（可能是 video0/1/2，取决于系统）。

**3. 列出视频采集设备详情**

```bash
# 若未安装：sudo apt update && sudo apt install -y v4l-utils
v4l2-ctl --list-devices
```

**4. 检查 USB 是否有报错（供电/枚举问题会在这里暴露）**

```bash
dmesg | grep -i usb | tail -n 30
```

留意是否有 `device descriptor read error`、`unable to enumerate`、`reset` 等字样——这些通常意味着**供电不足或线材差**。

**5. 检查是否发生过欠压 / 异常复位**

```bash
dmesg | grep -iE "under.?volt|brownout|reset|power" | tail -n 20
```

## 六、供电稳定性压力测试

确认硬件识别正常后，做一次"压力测试"，确保跑满负载也不断电。

**1. 实时看温度（RK3588 满载会发热，过热会降频甚至关机）**

```bash
watch -n 1 'cat /sys/class/thermal/thermal_zone*/temp'
```

数值除以 1000 即为摄氏度。持续 > 80℃ 要考虑散热（加风扇/散热片）。

**2. 看 CPU 频率调度策略**

```bash
cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
```

**3. 正式跑你们的检测程序，连续运行 ≥ 10 分钟**，同时另开一个终端盯 `dmesg -w` 看有没有 USB 掉线或欠压日志。

| 测试结果 | 判断 |
|---------|------|
| 连续运行不断电，摄像头不掉线 | ✅ 供电方案成功 |
| 启动后几秒~十几秒断电 | ⚠️ 峰值电流仍偏高，见第七、八节优化 |
| 摄像头反复重连 / 花屏 | ⚠️ Hub 供电不足，见第八节 |
| 根本开不了机 | ❌ PD 那一路有问题，见第八节 |

> 若手头有 **USB-C 双头电流表**，串在 C-C 线中间，跑程序时盯读数：电压稳在 4.9~5.1V、峰值电流 < 3A 即正常；电压跌破 4.7V 说明电源/线材拉胯。这组数据也能写进策划书"实测"部分。

## 七、软件侧优化（配合硬件，进一步稳）

**1. 摄像头索引可能变化**

接 Hub 后摄像头设备号可能不再是 0。用下面脚本探测，再改 `config.py`：

```python
import cv2
for i in range(5):
    cap = cv2.VideoCapture(i)
    if cap.isOpened():
        ret, frame = cap.read()
        if ret:
            print(f"可用摄像头: /dev/video{i}  分辨率={frame.shape}")
        cap.release()
```

**2. 错峰启动，避免 CPU 与摄像头同时拉满**

在 `main.py` 初始化序列里加延时：

```python
import time
time.sleep(3)        # 系统先稳定
init_camera()        # 再开摄像头
time.sleep(1)
init_mediapipe()     # 再加载模型
init_serial()        # 最后开串口 / LED
```

**3. 启动阶段降低瞬时功耗（可选）**

启动时先用较低分辨率，减小摄像头初始化电流冲击，稳定后再切回：

```python
# config.py 启动先用 720p，跑稳后再切 1080p
CAM_WIDTH, CAM_HEIGHT = 1280, 720
```

**4. 把 CPU 调度策略设为 ondemand（启动期更省电）**

```bash
echo ondemand | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
```

## 八、故障排查表

| 现象 | 可能原因 | 解决办法 |
|------|---------|---------|
| 一开机就断电 / 开不了机 | PD 协商失败，香橙派没拿到 5V/3A | 用**方案三**插入 PD 诱骗器强制 5V；或确认充电器支持 **5V/3A** 档 |
| 跑程序几秒后断电 | 启动峰值电流仍冲顶 | ① 摄像头务必走 Hub（方案一）② 在香橙派 5V 输入端并 **1000μF/16V 电解 + 10μF 陶瓷**电容吸收浪涌 ③ 降分辨率 + 错峰启动 |
| `lsusb` 看不到摄像头 | 上联线没插对 / 误插了 Type-C 供电口 / Hub 没供电 | Hub 上联线必须插**香橙派 USB-A 口**（不是 Type-C 供电口）；确认 Hub DC 口已通电、指示灯亮 |
| 摄像头能识别但花屏/卡顿/掉线 | Hub 单口供电不足 | 换 Hub 另一个下游口；确认 USB 转 DC 线接的是 **5V** 源；线材太细就换粗的 |
| 充电宝（移动电源）给 Hub 供电后自动断电 | 充电宝"小电流自动关机"保护 | ① 改用充电器供 Hub ② 选带"低电流/小电流模式"的充电宝 ③ Hub 上再插一个耗电设备凑电流 |
| `dmesg` 出现 USB 欠压/复位错误 | 线材压降大 / 电源弱 | 换更短更粗的线；换 PD 5V/3A 能力更强的电源 |
| 运行一会卡死、降频 | 过热 | 给香橙派加散热片/小风扇，别封死在密闭箱里 |

## 九、独立运行 / 车载注意事项

- **真正脱离插座**：把"大功率充电器"换成**支持 PD 的多口移动电源**，方案一/二的接法不变。
- **多余线缆处理**：0.5m 的线都很短，基本不用收纳；若做集成，多余线**折叠成 S 形**再用魔术贴扎带固定，**不要盘成圆圈**（盘圈会形成电感引入噪声，影响 NIR-rPPG 微弱信号）。
- **电源线与相机数据线分开走**，不要捆在一起，避免电源噪声串扰到摄像头数据，拉低你们的心率检测精度。
- **散热优先**：集成进箱体时务必留散热孔或装小风扇。
- **现阶段建议敞开式底板**（洞洞板/亚克力板）固定各模块，方便调试，封箱留到系统跑稳、答辩前再做。

## 十、最小可用配置（应急兜底）

如果到货不全、或 Hub/诱骗器调不通，**只要有 PD 电源 + C-C 线就能先跑起来**：

```
PD电源 ──C-C 100W线──▶ 香橙派 Type-C供电口
摄像头 ──直接插──▶ 香橙派 USB-A口
```

理由：供电已从 5V/2A 升级到 **PD 5V/3A**，香橙派 + 摄像头总电流约 2.5A，**仍在 3A 之内**，大概率能稳定运行。若偶尔断电，按第八节加 **1000μF 电容**即可。  
**先用这个最简配置验证系统能不能跑，再逐步加 Hub 做电源隔离。**

---

### 一句话总览

> **香橙派走 PD 5V/3A（C-C 线，必要时加诱骗器锁 5V）；摄像头走有源 Hub 独立供电（USB 转 DC 线）；数据用 Hub 上联线回插香橙派 USB 口。电源分两路、数据走一根，峰值电流不叠加，断电问题即解。**
