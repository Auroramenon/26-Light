# 香橙派 + 2.8寸 ILI9341 显示屏集成指南

> 适用屏幕：欣薇 2.8寸 SPI 彩屏（驱动 ILI9341，320×240，3.3-5V 兼容版本，带 SD 卡槽）
> 适用平台：香橙派系列（Orange Pi Zero 2W / Orange Pi 3B / Orange Pi 5 等带 40Pin GPIO 的型号）
> 目标：在 3D 打印外壳内嵌入小屏幕，实时显示疲劳等级、心率等参数

---

## 一、硬件清单

| 序号 | 元件 | 规格 | 数量 | 备注 |
|------|------|------|------|------|
| 1 | 显示屏 | 欣薇 2.8寸 ILI9341 SPI 屏，320×240，3.3-5V | 1 | 选 14 针带 SD 卡版 |
| 2 | 杜邦线 | 母对母，约 15cm | 8 根 | 推荐不同颜色便于区分 |
| 3 | 香橙派 | 已安装 Orange Pi OS / Armbian | 1 | 系统需开启 SPI |

---

## 二、屏幕引脚说明

欣薇 2.8寸 ILI9341 标准 14 针版引脚定义（PCB 上有丝印标注）：

| 引脚号 | 名称 | 类型 | 说明 |
|--------|------|------|------|
| 1 | VCC | 电源 | 3.3V 或 5V（板载 LDO 自动适配） |
| 2 | GND | 电源 | 地 |
| 3 | CS | 输入 | 显示屏片选（低有效） |
| 4 | RESET | 输入 | 显示屏复位 |
| 5 | DC/RS | 输入 | 数据/命令选择 |
| 6 | SDI(MOSI) | 输入 | SPI 主出从入 |
| 7 | SCK | 输入 | SPI 时钟 |
| 8 | LED | 电源 | 背光电源（3.3V 或 PWM 控制） |
| 9 | SDO(MISO) | 输出 | SPI 主入从出（仅 SD 卡或屏幕读取用） |
| 10 | T_CLK | 输入 | 触摸时钟（本项目不用） |
| 11 | T_CS | 输入 | 触摸片选（本项目不用） |
| 12 | T_DIN | 输入 | 触摸数据入（本项目不用） |
| 13 | T_DO | 输出 | 触摸数据出（本项目不用） |
| 14 | T_IRQ | 输出 | 触摸中断（本项目不用） |

**本项目只需接 8 根线**（VCC、GND、CS、RESET、DC、MOSI、SCK、LED），触摸相关 5 针全部悬空。

## 三、接线表（针对香橙派 40Pin GPIO）

### 3.1 接线总表

```
ILI9341 屏幕          香橙派 40Pin GPIO          物理引脚
──────────────       ─────────────────────      ────────
VCC      ──────────→ 3.3V                       Pin 1
GND      ──────────→ GND                        Pin 6
LED      ──────────→ 3.3V                       Pin 17（或共用 Pin 1）
SDI(MOSI)─────────→ SPI0_MOSI                  Pin 19
SCK      ──────────→ SPI0_CLK                   Pin 23
CS       ──────────→ SPI0_CS0                   Pin 24
DC/RS    ──────────→ GPIO（任意空闲，推荐GPIO25）Pin 22
RESET    ──────────→ GPIO（任意空闲，推荐GPIO27）Pin 13
SDO(MISO)──×── 不接
T_*  五针 ──×── 不接（触摸功能未使用）
```

### 3.2 香橙派 40Pin 物理引脚示意图

```
                 香橙派 40Pin GPIO
                 ┌─────────────────┐
            3.3V │ 1   2  │ 5V
                 │        │
        I2C_SDA  │ 3   4  │ 5V
        I2C_SCL  │ 5   6  │ GND      ←── 接 GND
                 │        │
          GPIO7  │ 7   8  │ TX
            GND  │ 9   10 │ RX
                 │        │
         GPIO17  │ 11  12 │ GPIO18
         GPIO27  │ 13  14 │ GND      ←── RESET 接 Pin 13
         GPIO22  │ 15  16 │ GPIO23
                 │        │
           3.3V  │ 17  18 │ GPIO24   ←── LED 接 Pin 17
            MOSI │ 19  20 │ GND      ←── MOSI 接 Pin 19
            MISO │ 21  22 │ GPIO25   ←── DC 接 Pin 22
            SCLK │ 23  24 │ CS0      ←── SCK 接 Pin 23, CS 接 Pin 24
                 │        │
            GND  │ 25  26 │ GPIO7
                 │  ...   │
                 └─────────────────┘
                 ↑
              VCC 接 Pin 1
```

## 四、香橙派系统配置

### 4.1 启用 SPI 接口

**方法一：图形配置工具（推荐）**

```bash
sudo orangepi-config
```

在菜单中选择：
- `System` → `Hardware`
- 找到 `spi-spidev` 并勾选（按空格键）
- 选择 `Save` → `Back` → `Exit`
- 重启系统：`sudo reboot`

**方法二：手动编辑配置文件（Armbian 系统）**

```bash
sudo nano /boot/armbianEnv.txt
```

在 `overlays=` 行末尾追加 `spi-spidev`，例如：

```
overlays=spi-spidev
param_spidev_spi_bus=0
```

保存后重启：

```bash
sudo reboot
```

### 4.2 验证 SPI 是否启用成功

```bash
ls /dev/spidev*
```

应看到类似输出：

```
/dev/spidev0.0  /dev/spidev0.1
```

如果没有 `spidev0.0`，说明 SPI 未正确启用，回到 4.1 步骤检查。

### 4.3 安装 Python 驱动库

```bash
# 进入项目 Python 虚拟环境（如有）
cd /path/to/26-Light/python
source venv/bin/activate  # 如使用 venv

# 安装屏幕驱动库
pip install luma.lcd luma.core Pillow

# 安装 GPIO 库（用于控制 DC/RESET 引脚）
pip install OPi.GPIO        # 香橙派专用
# 或使用通用库
pip install RPi.GPIO        # 部分香橙派系统兼容
```

### 4.4 添加用户权限

为避免每次运行需要 `sudo`，将当前用户加入 spi 和 gpio 组：

```bash
sudo usermod -aG spi,gpio $USER
# 退出重新登录使权限生效
```

### 5.3 新建文件：`python/display/oled_screen.py`

完整可直接使用的代码：

```python
"""
ILI9341 2.8寸 SPI 显示屏驱动
用于嵌入式部署时显示疲劳等级与生理参数
"""
from luma.core.interface.serial import spi
from luma.lcd.device import ili9341
from luma.core.render import canvas
from PIL import ImageFont
import os


# ===== 等级映射 =====
LEVEL_COLORS = {
    0: (0, 255, 0),       # 绿色 - 正常
    1: (255, 255, 0),     # 黄色 - 轻度疲劳
    2: (255, 128, 0),     # 橙色 - 中度疲劳
    3: (255, 0, 0),       # 红色 - 重度疲劳
}

LEVEL_NAMES = {
    0: "NORMAL",
    1: "MILD",
    2: "MODERATE",
    3: "SEVERE",
}


class OLEDScreen:
    """ILI9341 屏幕显示控制类"""

    def __init__(self, gpio_dc=25, gpio_rst=27, port=0, device=0):
        """
        初始化屏幕

        Args:
            gpio_dc: DC 引脚 GPIO 编号（对应物理 Pin 22）
            gpio_rst: RESET 引脚 GPIO 编号（对应物理 Pin 13）
            port: SPI port (默认 0)
            device: SPI device (默认 0, 对应 CS0)
        """
        serial_interface = spi(
            port=port,
            device=device,
            gpio_DC=gpio_dc,
            gpio_RST=gpio_rst,
        )
        self.device = ili9341(
            serial_interface,
            width=320,
            height=240,
            rotate=0,
        )

        # 加载字体（中文需要 TTF 字体文件）
        font_path = "/usr/share/fonts/truetype/dejavu/DejaVuSans-Bold.ttf"
        if os.path.exists(font_path):
            self.font_large = ImageFont.truetype(font_path, 32)
            self.font_medium = ImageFont.truetype(font_path, 22)
            self.font_small = ImageFont.truetype(font_path, 16)
        else:
            self.font_large = ImageFont.load_default()
            self.font_medium = ImageFont.load_default()
            self.font_small = ImageFont.load_default()

    def update(self, level: int, hr: int, score: float,
               perclos: float = 0.0, yawn_rate: float = 0.0):
        """
        刷新屏幕显示

        Args:
            level: 疲劳等级 (0/1/2/3)
            hr: 心率 BPM
            score: 综合疲劳评分 (0.0-1.0)
            perclos: 闭眼占比 (0.0-1.0)
            yawn_rate: 哈欠频率 (次/分钟)
        """
        color = LEVEL_COLORS.get(level, (255, 255, 255))
        name = LEVEL_NAMES.get(level, "UNKNOWN")

        with canvas(self.device) as draw:
            # 顶部彩色等级条 (320x60)
            draw.rectangle((0, 0, 320, 60), fill=color)
            draw.text((20, 12), f"LEVEL {level}",
                      font=self.font_large, fill="black")
            draw.text((150, 22), name,
                      font=self.font_medium, fill="black")

            # 中部参数区域
            y = 80
            draw.text((20, y),
                      f"HR     : {hr:3d} BPM",
                      font=self.font_medium, fill="white")
            draw.text((20, y + 35),
                      f"Score  : {score:.2f}",
                      font=self.font_medium, fill="white")
            draw.text((20, y + 70),
                      f"PERCLOS: {perclos * 100:5.1f} %",
                      font=self.font_medium, fill="white")
            draw.text((20, y + 105),
                      f"Yawn   : {yawn_rate:4.1f} /min",
                      font=self.font_medium, fill="white")

            # 底部状态条
            draw.rectangle((0, 220, 320, 240), fill=(40, 40, 40))
            draw.text((10, 222),
                      "Fatigue Monitor v1.0",
                      font=self.font_small, fill="gray")

    def clear(self):
        """清空屏幕"""
        with canvas(self.device) as draw:
            draw.rectangle(self.device.bounding_box, fill="black")

    def show_boot_screen(self):
        """显示开机画面"""
        with canvas(self.device) as draw:
            draw.rectangle(self.device.bounding_box, fill="black")
            draw.text((40, 80), "Fatigue System",
                      font=self.font_large, fill="cyan")
            draw.text((90, 130), "Booting...",
                      font=self.font_medium, fill="white")
            draw.text((30, 200), "Orange Pi + ILI9341",
                      font=self.font_small, fill="gray")
```

### 5.4 在 `main.py` 中的集成位置

> 以下仅为示意，**不要直接修改代码**，部署时参照即可。

**在 main.py 顶部导入区域添加：**

```python

```

**在初始化区域（与 serial_sender 同级位置）添加：**

```python
# 初始化小屏幕（嵌入式部署使用）
try:
    oled = OLEDScreen(gpio_dc=25, gpio_rst=27)
    oled.show_boot_screen()
except Exception as e:
    print(f"[WARN] 小屏幕初始化失败，跳过: {e}")
    oled = None
```

**在主循环里，每次输出疲劳等级时同步刷新屏幕：**

```python
# 原有发送串口的位置附近
if serial_sender is not None:
    serial_sender.send(level, hr)

# 新增：同步刷新小屏幕（约 1-2 秒一次即可）
if oled is not None:
    oled.update(
        level=level,
        hr=hr,
        score=fatigue_score,
        perclos=perclos_value,
        yawn_rate=yawn_per_min,
    )
```

### 5.5 屏幕刷新频率建议

- **不要每帧都刷新**：SPI 写整屏需要约 100ms，30fps 刷新会拖累主循环
- **推荐做法**：每秒刷新 1-2 次即可，与等级判定周期对齐
- **实现方法**：在 main.py 主循环中加计时器

```python
import time
last_oled_update = 0
OLED_UPDATE_INTERVAL = 0.5  # 秒

# 主循环内
now = time.time()
if oled is not None and (now - last_oled_update) >= OLED_UPDATE_INTERVAL:
    oled.update(level, hr, score, perclos, yawn_rate)
    last_oled_update = now
```

**示例 3：添加心率波形图（小型 BVP）**

```python
def update_with_bvp(self, level, hr, bvp_buffer):
    """bvp_buffer: 最近 N 个 BVP 采样点的 list"""
    color = LEVEL_COLORS.get(level, (255, 255, 255))
    with canvas(self.device) as draw:
        # 顶部信息
        draw.rectangle((0, 0, 320, 50), fill=color)
        draw.text((20, 10), f"Lv {level}  HR {hr}",
                  font=self.font_large, fill="black")

        # 中部波形（160x100 区域）
        if len(bvp_buffer) > 1:
            w, h = 280, 100
            x0, y0 = 20, 80
            n = len(bvp_buffer)
            bmin, bmax = min(bvp_buffer), max(bvp_buffer)
            span = (bmax - bmin) or 1
            pts = [
                (x0 + int(i * w / (n - 1)),
                 y0 + h - int((v - bmin) * h / span))
                for i, v in enumerate(bvp_buffer)
            ]
            draw.line(pts, fill="lime", width=2)
```

## 八、故障排查

### 8.1 屏幕完全不亮

| 可能原因 | 排查方法 |
|---------|---------|
| 电源未通 | 万用表测 VCC 和 GND 间是否有 3.3V |
| 背光未通 | 检查 LED 引脚是否接 3.3V |
| 接线松动 | 重新插拔所有杜邦线 |
| SPI 未启用 | `ls /dev/spidev*` 应能看到设备 |

### 8.2 屏幕亮但显示乱码/雪花点

| 可能原因 | 排查方法 |
|---------|---------|
| MOSI/SCK 接错 | 用万用表蜂鸣档复查接线 |
| DC/RST GPIO 编号错 | 确认 `gpio_DC` `gpio_RST` 参数对应物理引脚 |
| SPI 速率过高 | 在 `spi()` 中加 `bus_speed_hz=24000000` 降速 |
| 屏幕反向 | 修改 `rotate` 参数 |

### 8.3 颜色显示反色（红显蓝、蓝显红）

ILI9341 有部分批次 RGB/BGR 顺序不同，修改：

```python
self.device = ili9341(serial_interface, width=320, height=240,
                      rotate=0, bgr=True)   # 切换为 BGR
```

### 8.4 中文显示成方块

字体未加载中文字符集，下载并使用支持中文的 TTF：

```bash
# 下载思源黑体到香橙派
wget https://github.com/adobe-fonts/source-han-sans/raw/release/OTF/SimplifiedChinese/SourceHanSansSC-Regular.otf
sudo mv SourceHanSansSC-Regular.otf /usr/share/fonts/
```

然后在 `oled_screen.py` 中指向新字体路径。

### 8.5 屏幕刷新卡顿

- 降低刷新频率：`oled_update_interval` 改为 1.0 秒
- 简化绘制内容：减少 `draw.text()` 调用次数
- 提高 SPI 速率：`bus_speed_hz=32000000`

## 附录 A：完整 GPIO 编号对照表

香橙派 GPIO 编号方式有两种，本指南使用的是**香橙派内核 GPIO 编号**（非 BCM 编号）。

| 物理引脚 | 香橙派 GPIO | wPi 编号 | 功能复用 |
|---------|------------|---------|---------|
| Pin 1 | - | - | 3.3V |
| Pin 6 | - | - | GND |
| Pin 13 | GPIO27 | 2 | 通用 IO（本项目用于 RESET） |
| Pin 17 | - | - | 3.3V |
| Pin 19 | GPIO10 | 12 | SPI0_MOSI |
| Pin 22 | GPIO25 | 6 | 通用 IO（本项目用于 DC） |
| Pin 23 | GPIO11 | 14 | SPI0_CLK |
| Pin 24 | GPIO8 | 10 | SPI0_CS0 |

> **不同型号香橙派 GPIO 编号略有差异**，部署时请先用 `gpio readall` 命令查看本机对应关系，必要时调整 `oled_screen.py` 中的 `gpio_dc` 和 `gpio_rst` 参数。
